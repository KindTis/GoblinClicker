# 고블린 클릭커 웹 게임 스펙

## 1. 개요

고블린 클릭커는 직접 클릭 중심의 액티브 클릭커 게임이다. 플레이어는 화면 중앙의 고블린을 클릭해 체력을 깎고, 체력이 0이 되면 고블린을 무찌른 수와 재화를 얻는다. 재화는 클릭 피해량과 보조 효과를 강화하는 도구를 구매하는 데 사용한다.

첫 버전의 목표는 짧은 세션 안에서 다음 흐름이 명확하게 반복되는 MVP를 만드는 것이다.

1. 고블린을 클릭한다.
2. 고블린 체력이 감소한다.
3. 체력이 0이 되면 고블린을 무찌른 것으로 처리한다.
4. 재화를 얻고 다음 고블린이 등장한다.
5. 재화로 업그레이드를 구매한다.
6. 더 강한 고블린을 더 빠르게 무찌른다.

## 2. 핵심 가정

- 플랫폼은 데스크톱과 모바일 브라우저다.
- 첫 버전은 싱글 플레이이며 서버, 계정, 랭킹은 포함하지 않는다.
- 저장 데이터는 브라우저 `localStorage`에 보관한다.
- 게임은 세로형 모바일 화면에서도 플레이 가능해야 한다.
- 전투 실패, 플레이어 체력, 시간 제한은 MVP 범위에 포함하지 않는다.
- 자동 피해는 클릭 피로를 줄이고 처치 직전 마무리를 돕는 보조 수단이며, 핵심 재미와 가장 빠른 진행 방법은 직접 클릭에 둔다.
- 게임은 명시적인 엔딩이나 레벨 상한 없이 고블린 레벨과 처치 수가 계속 증가하는 무한 진행 구조로 둔다.

## 3. 기술 스택

### 권장 스택

- 언어: TypeScript
- 빌드 도구: Vite
- 게임 런타임: Phaser 4.x
- UI: HTML/CSS DOM 오버레이
- 테스트: Vitest, Playwright
- 저장: `localStorage`
- 배포: 정적 사이트 배포 방식

### 선택 이유

- Phaser는 2D HTML5 게임에 맞는 장면, 입력, 애니메이션, 사운드, 파티클 기능을 제공한다.
- Vite는 TypeScript 기반 웹 프로젝트를 빠르게 실행하고 정적 빌드로 배포하기 쉽다.
- HUD, 상점, 설정 같은 텍스트 중심 UI는 DOM으로 만들면 접근성과 반응형 처리가 쉽다.
- 게임 규칙은 Phaser 장면 내부에 직접 섞지 않고 순수 TypeScript 모듈로 분리한다. 이렇게 하면 체력, 보상, 업그레이드 계산을 Vitest로 검증할 수 있다.
- Playwright는 실제 브라우저에서 클릭, 구매, 저장 복원 흐름을 검증하는 데 사용한다.

### 최소 실행 환경

- Node.js 22.x LTS 권장
- 최신 Chromium, Firefox, Safari 계열 브라우저
- 터치 입력과 마우스 입력 모두 지원

## 4. 화면 구성

### 메인 전투 화면

- 중앙: 현재 고블린 스프라이트
- 고블린 하단: 체력 바와 현재 체력 숫자
- 상단: 처치 수, 보유 재화, 투석기 보유 시 투석기 쿨다운과 다음 자동 공격 피해량
- 데스크톱 우측: 업그레이드 상점 상시 표시
- 모바일 하단: 접힌 상점 버튼
- 좌측 또는 하단: 현재 클릭 피해량, 현재 고블린 레벨

HUD의 저장 경고는 현재 세션의 최신 `SaveData`가 저장소에 반영되지 않았다는 뜻의 비차단 경고다. 일반 저장 실패에서는 `저장되지 않음`으로 마지막 저장 시도 실패를 표시하고, 저장소 접근 실패로 인한 비저장 세션에서는 `저장소에 접근할 수 없어 저장되지 않음`처럼 더 구체적으로 표시한다. 로드 오류나 저장 데이터 삭제 실패 같은 차단 오류를 대신 표시하지 않는다.

모바일에서는 고블린과 체력 바를 먼저 보여주고, 상점은 기본 접힘 상태로 시작한다. 하단 고정 `상점` 버튼으로 열고 닫는다.

처치 후 300ms 전환 중에는 전역 상태 표시와 적 전용 표시를 분리한다. 처치 수, 보유 재화, 저장 실패 경고 같은 전역 상태 표시는 처치 판정 직후 즉시 갱신한다. 상점의 구매 가능 상태도 보유 재화 증가를 반영해 즉시 갱신할 수 있지만, 전환 중 구매 버튼은 입력 차단 상태를 유지한다. 고블린 레벨 표시, 고블린 체력 바, 현재 고블린 체력 숫자 같은 적 전용 표시는 300ms 동안 처치된 고블린의 상태를 유지한다. 체력 바는 0 또는 처치 상태로 보이며, 다음 고블린이 화면에 나타나는 시점에 고블린 레벨, 최대 체력, 현재 체력 바를 새 고블린 값으로 갱신한다.

### 상점 화면

상점은 별도 페이지가 아니라 같은 화면의 패널로 제공한다.

- 구매 가능한 업그레이드 목록
- 각 업그레이드의 현재 레벨
- 다음 1레벨 비용
- 다음 1레벨 효과
- 구매 가능 여부

상점의 업그레이드 행 표시 순서는 `낡은 몽둥이` -> `삐걱대는 투석기` -> `고블린 미끼 주머니` -> `진흙 함정`으로 고정한다. 구매 가능 여부, 추천 상태, 비용 부족 상태가 바뀌어도 행 순서는 바뀌지 않는다. 구매 가능 항목을 자동으로 위로 올리지 않고, 추천 항목 재배치, 동적 정렬, 자동 상단 이동은 MVP에 포함하지 않는다. 구매 가능 여부는 행 위치가 아니라 강조와 상태 문구로만 표시한다.

MVP의 업그레이드 구매는 한 번에 1레벨 구매만 지원한다.

업그레이드 구매 입력 1회는 해당 업그레이드의 1레벨 구매 시도 1회로 처리한다. 구매 입력은 항상 최신 메모리 상태 기준으로 순차 처리한다. 첫 구매가 성공하면 즉시 재화를 차감하고 레벨을 올린 뒤 다음 비용, 다음 효과, 구매 가능 상태를 갱신한다. 그다음 클릭, 탭, Enter, Space 활성화가 새로 들어오면 갱신된 재화와 갱신된 다음 비용 기준으로 다시 구매 가능 여부를 판단한다. 플레이어가 실제로 두 번 입력했고 두 번째 비용도 감당 가능하면 2레벨 구매가 가능하지만, 첫 구매 후 재화가 부족하면 두 번째 입력은 비용 부족 실패 피드백으로 처리한다. 같은 입력 이벤트를 중복 구매로 처리하지 않는다.

MVP 업그레이드 4종은 모두 상점에 표시한다. 비용이 부족한 업그레이드는 숨기거나 잠그지 않고, 비용 부족 상태와 다음 목표로 남은 비용을 명확히 보여준다. 비용 부족 버튼은 실제 HTML `disabled`로 만들지 않고, `구매 불가` 상태로 표시하되 포커스 가능해야 한다. 접근성 측면에서 비용 부족 상태는 `녹슨 동전 7개 부족`처럼 텍스트로도 드러나야 한다. 첫 구매 시점에는 `낡은 몽둥이`만 구매 가능하도록 경제가 설계되어 있다. `낡은 몽둥이` 레벨이 0인 동안만 작은 `추천` 표시를 보여주며, 비용 부족 상태여도 레벨 0이면 추천 표시는 유지할 수 있다. 첫 구매로 `낡은 몽둥이` 레벨이 1이 되면 추천 표시는 제거한다. 다른 업그레이드에는 MVP에서 추천 표시를 주지 않는다. 별도 튜토리얼 팝업은 MVP에 포함하지 않는다.

비용이 부족한 업그레이드 버튼을 클릭, 탭, Space, Enter로 활성화하면 구매를 실행하지 않고 게임 상태를 변경하지 않는다. 재화, 업그레이드 레벨, 투석기 쿨다운, 저장 데이터는 변경하지 않으며 저장 트리거도 발생시키지 않는다. 실패 피드백은 해당 버튼에만 짧게 표시한다. 예시는 버튼 흔들림, 비용 텍스트 강조, `녹슨 동전 부족` 문구다. 전역 모달이나 토스트 큐는 MVP에 포함하지 않는다.

업그레이드 구매에 성공하면 재화를 즉시 차감하고 해당 업그레이드 레벨을 1 올린다. 다음 효과, 다음 비용, 보유 재화, 관련 능력치 표시는 즉시 갱신한다. 성공 피드백은 버튼 눌림, 레벨 숫자 증가 강조, 재화 숫자 갱신, 다음 효과와 다음 비용 갱신 같은 버튼과 숫자 중심의 짧은 로컬 피드백으로 제한한다. 전역 축하 모달, 화면 전체 연출, 긴 튜토리얼 안내는 MVP에 포함하지 않는다.

각 업그레이드 행에는 상태 문구 영역을 고정 높이로 예약한다. `잠시 후 구매 가능`, `녹슨 동전 N개 부족`, 구매 가능 상태, 구매 성공 피드백, 구매 실패 피드백이 바뀌어도 업그레이드 행의 전체 높이는 변하지 않아야 한다. 상태 문구는 예약된 영역 안에서만 바뀌며, 전환, 재화 갱신, 구매 가능 상태 변화가 발생해도 특정 업그레이드 행으로 자동 스크롤하지 않는다.

저장소 접근 실패로 인한 비저장 세션에서도 업그레이드 구매 성공 처리는 메모리 상태에서 정상 수행한다. 재화 차감, 업그레이드 레벨 증가, 다음 효과와 다음 비용 재계산, 관련 능력치 갱신은 일반 세션과 동일하게 처리한다. 다만 저장 트리거는 실제 `goblin-clicker.save` 쓰기를 수행하지 않고 저장 실패 상태로 남기며, HUD의 저장소 접근 실패 경고를 유지한다.

데스크톱에서는 상점을 우측 패널로 상시 표시한다. 모바일에서는 상점을 하단 시트로 표시하며, 기본 최대 높이는 `45dvh`로 둔다. 시트 내용이 넘치면 내부 스크롤로 처리한다. 단, 상점이 열린 상태에서도 직접 공격 영역은 최소 160px 이상 보여야 하며 체력 바는 가려지지 않아야 한다. 작은 화면에서 `45dvh`와 직접 공격 영역 보존이 충돌하면 직접 공격 영역 보존을 우선하고 시트 높이를 더 줄인다. 구매 후에도 상점은 열린 상태를 유지한다. 상점이 열린 상태에서 직접 공격이나 투석기 자동 공격으로 300ms 처치 전환이 시작되어도 모바일 하단 시트의 열린 상태와 내부 스크롤 위치는 초기화하지 않는다. 모바일 하단 시트의 스크롤 컨테이너에는 `padding-bottom`과 `scroll-padding-bottom`을 모두 `하단 상점 토글 높이 + safe-area-inset-bottom + 8px` 이상으로 둔다. 마지막 업그레이드 행과 마지막 포커스 가능 요소는 스크롤 가능한 최하단 위치에서 하단 `상점` 토글 버튼과 안전 영역 위로 완전히 올라와야 한다.

상점이 열려도 게임은 일시정지하지 않는다.

300ms 처치 전환 중에도 상점은 보일 수 있지만 업그레이드 구매 버튼은 일시적으로 입력 차단 상태로 둔다. 키보드 포커스를 가진 구매 버튼은 가능하면 포커스를 유지한다. 이를 위해 실제 HTML `disabled`로 포커스를 떨어뜨리기보다 `aria-disabled` 또는 동등한 상태 표시와 구매 핸들러 차단을 우선한다. 전환 중 Enter, Space, 클릭, 탭이 들어와도 구매 입력은 처리하지 않고 큐에 쌓지 않는다. 전환 중 구매 버튼에는 `잠시 후 구매 가능` 또는 `처치 처리 중` 같은 짧은 버튼 단위 상태 텍스트를 노출한다. 이 문구는 시각적으로 버튼 하단 또는 비용/효과 영역에 작게 표시하거나, 최소한 `aria-describedby`로 버튼에 연결한다. 전역 토스트, 모달, 화면 전체 알림은 사용하지 않는다. 전환 입력 차단은 비용 부족 버튼의 실패 피드백 규칙보다 우선하므로, 전환 중에는 `녹슨 동전 부족` 같은 비용 부족 문구보다 `잠시 후 구매 가능` 같은 전환 상태 문구를 표시한다. 다음 고블린이 화면에 표시된 뒤 구매 버튼을 다시 활성화하고, 평상시 구매 가능 또는 구매 불가 상태 텍스트로 돌아간다. 구현상 실제 `disabled`를 써서 포커스가 사라지는 경우에도, 전환 종료 후 자동 구매나 입력 재시도는 하지 않는다.

300ms 처치 전환이나 로드 오류처럼 구매 버튼이 실제 입력 차단 상태인 경우에는 구매 성공 트랜잭션도, 구매 실패 트랜잭션도 시작하지 않는다.

전환 종료 시 같은 구매 버튼에 포커스가 남아 있어도 전환 중 들어온 Enter, Space, 클릭, 탭을 자동 재실행하지 않는다. 플레이어가 전환 종료 후 새 입력을 다시 해야 구매가 시도된다.

모바일에서 상점이 접혀 있을 때 실제 구매 가능한 업그레이드가 하나 이상 있으면 하단 `상점` 버튼에 `상점 1`, `상점 2`처럼 구매 가능 개수 배지를 표시한다. 구매 가능 개수는 `보유 재화 >= 다음 1레벨 비용`인 업그레이드 수로 계산한다. 구매 가능 개수가 0이면 배지를 숨긴다. 강조색은 보조 표현으로만 사용할 수 있으며, 색상만으로 구매 가능 상태를 전달하지 않는다. `낡은 몽둥이`가 레벨 0이라 `추천` 상태여도 재화가 부족하면 접힌 `상점` 버튼의 구매 가능 표시에는 포함하지 않는다. 전환 중 구매 입력 차단 상태에서는 구매 가능 개수를 보여줄 수 있지만, 버튼 상태 문구는 전환 상태를 우선한다. `추천`은 온보딩 힌트이고, 접힌 상점 버튼의 구매 가능 표시는 실제 구매 가능성이다.

모바일 하단 `상점` 버튼은 상점 시트를 여닫는 같은 토글 버튼을 유지한다. 이 버튼은 상점 시트와 연결된 `aria-controls`를 가진다. 상점이 닫혀 있으면 `aria-expanded="false"`, 열려 있으면 `aria-expanded="true"`를 사용한다. 닫힌 상태의 접근성 이름은 `상점 열기, 구매 가능 업그레이드 N개` 또는 구매 가능 개수가 0이면 `상점 열기, 구매 가능 업그레이드 없음`으로 둔다. 열린 상태의 접근성 이름은 `상점 닫기, 구매 가능 업그레이드 N개` 또는 `상점 닫기, 구매 가능 업그레이드 없음`으로 둔다.

모바일 상점을 열면 시트 컨테이너 또는 시트 제목에 일시적으로 포커스를 준다. 이 요소는 `tabindex="-1"` 같은 방식으로 프로그램 포커스만 받을 수 있게 하며, 시트 컨테이너나 제목에 포커스가 있는 상태에서 Enter 또는 Space가 구매나 닫기 동작을 실행하면 안 된다. 이후 Tab 이동으로 시트 내부 `닫기` 버튼과 업그레이드 버튼에 접근한다. 상점은 모달이 아니므로 시트 내부에 Tab 포커스 트랩을 만들지 않는다.

모바일 상점이 열려 있는 동안에도 하단 `상점` 토글 버튼은 계속 노출한다. 시트 내부 `닫기` 버튼은 상점 시트 상단 우측에 둔다. 시트 내부 `닫기` 버튼과 하단 `상점` 토글 버튼은 같은 닫기 동작을 수행한다. 모바일 하단 시트는 시트 내부 `닫기` 버튼, 하단 고정 `상점` 버튼 토글, 키보드 `S`로만 닫는다. 시트 바깥 탭으로 닫기는 지원하지 않는다. 스와이프 다운 닫기는 내부 스크롤과 제스처 충돌을 피하기 위해 MVP에 포함하지 않는다.

투석기를 구매한 뒤에는 자동 피해를 초당 피해량으로 환산하지 않고 `투석기: 3.2초 후 6 피해`처럼 다음 자동 공격까지 남은 시간과 다음 자동 공격 피해량으로 표시한다. 투석기 레벨이 0이면 HUD에 투석기 상태를 표시하지 않는다.

## 5. 게임 플레이 흐름

### 시작 상태

- 처치 수: 0
- 현재 고블린 레벨: `defeatedCount + 1 = 1`
- 현재 고블린 최대 체력: 5
- 현재 고블린 남은 체력: 5
- 클릭 기본 피해량: 1
- 진흙 함정 준비 레벨: 0
- 삐걱대는 투석기 레벨: 0
- 투석기 쿨다운 남은 시간: 0ms
- 보유 재화: 0

저장 데이터가 없는 첫 실행은 `localStorage` 읽기에 성공했지만 `goblin-clicker.save` 키가 없는 경우를 뜻한다. 이 경우 위 시작 상태의 `GameState`와 `SaveData`를 생성한 직후 `goblin-clicker.save`에 즉시 저장을 시도한다. 저장에 성공하면 경고 없이 게임을 시작하고, 저장에 실패해도 게임은 시작하되 HUD에 `저장되지 않음` 경고를 표시한다. 이후 다음 저장 트리거에서 다시 저장을 시도한다.

### 직접 공격 처리

1. 플레이어가 직접 공격을 입력한다.
2. 마지막 직접 공격 처리 후 80ms 안에 들어온 입력이면 무시한다.
3. `mudTrapArmedLevel`이 1 이상이라면 이번 직접 공격 피해에 진흙 함정 배율을 적용하고 `mudTrapArmedLevel`을 `0`으로 소모한다.
4. 계산된 직접 공격 피해량만큼 고블린 체력을 감소시키되, `GameState.goblinHp = max(0, goblinHp - damage)`로 0 이하를 클램프한다.
5. 피격 애니메이션과 타격 숫자 표시를 재생한다.
6. 클램프된 체력이 0인지 확인한다.
7. 0이라면 처치 처리로 넘어간다.

하나의 직접 공격 입력 처리 안에서 여러 저장 사유가 발생하면 중간 상태를 저장하지 않고, 입력 처리 후 최종 `SaveData`만 1회 저장한다. 직접 공격이 `진흙 함정`을 소모했지만 고블린이 살아 있으면 `mudTrapArmedLevel`이 `0`이 된 현재 전투 상태를 1회 저장한다. 같은 직접 공격으로 `진흙 함정`을 소모하고 고블린까지 처치하면, 처치 처리로 다음 고블린 상태까지 확정한 뒤 최종 상태를 1회만 저장한다.

런타임 내부에서 피해 처리 중 `goblinHp`가 0이 되는 것은 가능하지만, 이 상태는 즉시 처치 처리로 이어지는 일시 상태다. 현재 고블린의 0 체력 상태를 저장 가능한 전투 상태로 남기지 않는다.

직접 공격은 고블린 중심의 보이지 않는 고정 직접 공격 영역 안에서 시작한 마우스 또는 터치 `pointerdown`, 또는 게임 루트나 전투 화면 포커스 상태의 Space `keydown`을 뜻한다. 같은 포인터 조작에서 뒤따라오는 `click` 이벤트는 고블린 공격으로 처리하지 않는다. 직접 공격 영역 밖에서 시작한 포인터가 영역 안으로 들어와도 공격하지 않으며, 직접 공격 영역 안에서 시작한 `pointerdown`만 80ms 입력 제한을 거쳐 직접 공격 후보가 된다. Space `keyup`은 공격으로 처리하지 않고, `event.repeat`이 `true`인 Space `keydown`도 무시한다. 전투 포커스 상태에서 Space `keydown`을 직접 공격 후보로 처리할 때는 `preventDefault()`로 페이지 스크롤을 막는다. 직접 공격 입력 간 최소 간격은 80ms이며, 이는 오토클릭 방지가 아니라 모바일과 키보드 중복 이벤트를 막는 안정장치다. 투석기 자동 공격은 직접 공격이 아니며, `진흙 함정` 보너스를 받거나 소모하지 않는다.

한 프레임에서 직접 공격 입력과 투석기 자동 공격 만료가 모두 처리 가능하면 `직접 공격 입력 처리 -> 처치/전환 여부 확인 -> 투석기 쿨다운 갱신 및 자동 공격 처리` 순서로 처리한다. 직접 공격으로 고블린이 처치되면 즉시 처치 처리와 300ms 전환에 들어가며, 해당 프레임에서는 투석기 쿨다운을 차감하지 않고 자동 공격도 발사하지 않으며 다음 고블린에게 지연 예약하지 않는다. 직접 공격 후 고블린이 살아 있고 전환 상태가 아니라면 그다음 투석기 쿨다운을 갱신하고, 만료 시 자동 공격을 처리한다.

저장소 접근 실패로 인한 비저장 세션에서도 고블린 공격, 체력 감소, 처치, 재화 획득, 다음 고블린 상태 확정은 메모리 상태에서 정상 처리한다. 저장 트리거만 실제 쓰기를 수행하지 않으며, 저장소 접근 실패 경고는 계속 표시한다. 비저장 세션의 진행은 새로고침하면 사라질 수 있음을 HUD에서 명확히 알려야 한다.

직접 공격 영역은 고블린 스프라이트의 실제 픽셀 모양을 판정으로 쓰지 않는다. 영역은 고블린 스프라이트 중심에 고정하며 레벨, 장식, 피격 애니메이션, squash/stretch와 무관하게 유지한다. 모바일에서는 최소 `160px x 160px` 이상이어야 하며, 스프라이트가 160px보다 크게 보이는 화면에서는 보이는 주요 몸통보다 작아지지 않게 한다.

직접 공격 피해는 현재 고블린에게만 적용한다. `GameState.goblinHp`는 음수가 될 수 없으며, 체력이 0이 되면 초과 피해는 버리고 다음 고블린에게 이월하지 않는다.

### 투석기 자동 공격 처리

투석기 쿨다운은 `삐걱대는 투석기` 레벨이 1 이상이고, `document.visibilityState === "visible"`이며, 전투 중인 고블린이 표시된 동안 `performance.now()` 기반 단조 증가 시간의 delta로 흐른다. `document.visibilityState === "visible"`이면 브라우저 창 포커스가 없어도 게임 화면 활성 상태로 본다. `Date.now()`나 시스템 시각은 투석기 쿨다운, 처치 보상, 체력 계산에 사용하지 않는다. 투석기 레벨이 0이면 `catapultCooldownRemainingMs`는 `0`이며 쿨다운은 진행하지 않는다. 플레이어가 직접 공격하지 않아도 쿨다운은 감소한다.

프레임 내에서 직접 공격 입력이 먼저 고블린을 처치하지 않았고 전환 상태도 아니라면, 투석기 쿨다운과 자동 공격을 다음 순서로 처리한다.

1. `catapultCooldownRemainingMs`에서 `document.visibilityState === "visible"`인 동안의 `performance.now()` 기반 delta를 차감한다.
2. 쿨다운이 0 이하가 되면 투석기 자동 공격을 1회 적용한다.
3. 자동 공격 후 `catapultCooldownRemainingMs`를 `5000`으로 재설정한다.

프레임 경과 시간이 남은 쿨다운을 초과해도 초과 시간은 다음 쿨다운에 이월하지 않는다. 예를 들어 `catapultCooldownRemainingMs = 100`이고 프레임 경과 시간이 180ms이면 투석기 자동 공격을 1회 처리한 뒤 `catapultCooldownRemainingMs`를 정확히 `5000`으로 재설정하고, 초과된 80ms는 버린다. 어떤 경우에도 한 번의 쿨다운 갱신에서 투석기 자동 공격을 여러 발 처리하지 않는다.

`document.visibilityState === "hidden"`이면 투석기 쿨다운을 멈춘다. 다시 `visible`이 되었을 때 hidden 동안의 경과 시간을 catch-up하지 않도록 `performance.now()` 기준 시간을 새로 잡고, 여러 발을 몰아서 발사하지 않는다. 재개 시 쿨다운이 이미 만료된 상태라면 최대 1회만 처리하고, 이후 쿨다운을 다시 `5000`으로 둔다. 페이지가 닫혀 있거나 저장을 불러오는 동안 경과한 시간도 반영하지 않는다.

상점이 열려 있는 동안에도 `document.visibilityState === "visible"`이면 투석기 쿨다운과 자동 공격은 실제 경과 시간 기준으로 계속 진행된다.

처치 후 다음 고블린이 표시되기 전 300ms 전환 시간 동안에는 visibility와 무관하게 투석기 쿨다운을 차감하지 않는다. 전환 중 자동 공격은 발사하거나 지연 예약하지 않으며, 다음 고블린이 표시된 뒤 남아 있는 쿨다운 값으로 다시 진행한다. 로드 오류 상태도 visibility와 무관하게 투석기 쿨다운이 멈추는 별도 상태다.

투석기 자동 공격 피해도 현재 고블린에게만 적용하고, `GameState.goblinHp = max(0, goblinHp - damage)`로 0 이하를 클램프한다. 투석기 자동 공격으로 고블린 체력이 0이 되면 초과 피해는 버리고 직접 공격 처치와 동일한 처치 처리로 넘어간다. 공격 출처에 따른 보상 차등은 MVP에 넣지 않는다.

투석기 자동 공격으로 처치가 발생하면 처치 수, 재화, 다음 고블린 상태, 투석기 쿨다운까지 확정한 뒤 저장 사유를 기록한다. 같은 업데이트 흐름에서 10초 자동 저장 타이머도 도착했다면 별도 중간 저장을 하지 않고, 모든 처리가 끝난 뒤 최종 `SaveData`를 1회 저장한다.

### 처치 처리

처치 처리는 직접 공격과 투석기 자동 공격에 동일하게 적용한다.

오버킬 피해는 처치 처리에서 다음 고블린에게 이월하지 않는다. 다음 고블린은 계산된 최대 체력과 현재 체력으로 새로 등장한다.

1. 처치 전 `currentGoblinLevel = defeatedCount + 1` 기준 기본 처치 보상과 `고블린 미끼 주머니` 보너스를 합산한 재화를 지급한다.
2. 처치 수 `defeatedCount`를 1 증가시킨다.
3. 증가한 `defeatedCount` 기준으로 다음 `currentGoblinLevel = defeatedCount + 1`을 계산한다.
4. 다음 `currentGoblinMaxHp = calculateGoblinMaxHp(currentGoblinLevel)`를 계산하고 `goblinHp`를 이 값으로 초기화한다.
5. 진흙 함정 레벨이 1 이상이면 `mudTrapArmedLevel`을 현재 진흙 함정 레벨로, 아니면 `0`으로 초기화한다.
6. 투석기 레벨이 1 이상이면 `catapultCooldownRemainingMs`를 `5000`으로 초기화한다. 투석기 레벨이 0이면 `catapultCooldownRemainingMs`를 `0`으로 둔다.
7. 확정된 다음 고블린 상태를 기준으로 저장 트리거를 즉시 발생시킨다.
8. 300ms 처치 전환 시간 동안 짧은 처치 애니메이션 또는 파티클을 재생한다.
9. 전환 중 직접 공격 입력은 무시한다.
10. 전환 중 투석기 쿨다운은 차감하지 않고 자동 공격도 발사하지 않는다.
11. 전환 중 업그레이드 구매 입력은 처리하지 않고 큐에 쌓지 않는다.
12. 전환 중 상점의 열린 상태와 모바일 하단 시트 내부 스크롤 위치는 변경하지 않는다.
13. 300ms 뒤 다음 고블린을 화면에 표시하고 구매 버튼을 다시 활성화한다.

처치 직후 `GameState`는 이미 다음 고블린 상태로 확정되어 있지만, 렌더러와 HUD는 300ms 동안 처치 연출 스냅샷을 보여준다. 이 동안 처치 수, 보유 재화, 저장 실패 경고 같은 전역 상태 표시는 즉시 갱신된 값을 보여주고, 고블린 레벨, 체력 바, 현재 체력 숫자 같은 적 전용 표시는 처치된 고블린의 0 체력 또는 처치 상태를 유지한다.

정상 저장 플로우에서는 처치가 발생하면 다음 고블린 상태까지 확정한 뒤 저장하므로, 저장된 현재 고블린의 `goblinHp`가 0일 이유가 없다. 0 체력 표시는 300ms 처치 전환 중 렌더러가 보여주는 스냅샷일 뿐 저장 복원 대상 상태가 아니다.

### 진행 공식

MVP에서는 단순하고 예측 가능한 공식을 사용한다.

```text
현재 고블린 레벨 = defeatedCount + 1
현재 고블린 최대 체력 = floor(5 * 1.18 ^ (현재 고블린 레벨 - 1))
기본 처치 보상 = 1 + floor(현재 고블린 레벨 * 0.6)
처치 보상 = 기본 처치 보상 + 고블린 미끼 주머니 레벨
업그레이드 비용 = floor(기본 비용 * 성장률 ^ 현재 레벨)
현재 클릭 피해량 = 1 + 낡은 몽둥이 레벨
진흙 함정 첫 직접 공격 배율 = 1 + 2 * mudTrapArmedLevel
투석기 자동 공격 피해량 = 현재 클릭 피해량 * (1 + 삐걱대는 투석기 레벨)
투석기 쿨다운 = 5000ms
```

업그레이드 비용 공식의 `현재 레벨`은 구매 전 레벨이다. 모든 업그레이드는 레벨 0에서 시작하며, 첫 구매 비용은 기본 비용과 같다.

예시:

| 현재 고블린 레벨 | 최대 체력 | 처치 보상 |
| --- | ---: | ---: |
| 1 | 5 | 1 |
| 2 | 5 | 2 |
| 3 | 6 | 2 |
| 5 | 9 | 4 |
| 10 | 22 | 7 |

투석기의 `현재 클릭 피해량`은 `낡은 몽둥이`를 반영한 기본 직접 공격 피해량이며, `진흙 함정`의 첫 직접 공격 배율은 포함하지 않는다.

위 예시의 처치 보상은 `고블린 미끼 주머니` 효과를 제외한 기본 처치 보상이다. 현재 고블린 레벨은 저장하지 않고 `defeatedCount + 1`로 계산한다. 초기 구간은 빠르게 처치되도록 하고, 업그레이드를 사지 않으면 점점 클릭 수가 부담되도록 만든다.

## 6. 재화

재화 이름은 `녹슨 동전`으로 한다.

- 고블린 처치 시 획득한다.
- 업그레이드 구매에 사용한다.
- MVP에서는 재화를 잃는 이벤트를 만들지 않는다.
- 화면에는 보유량을 정수로 표시한다.

추후 확장용 보조 재화는 만들지 않는다. 첫 버전은 한 종류의 재화만 사용한다.

## 7. 업그레이드 도구

MVP 업그레이드는 4종으로 고정한다. 업그레이드는 모두 레벨 0에서 시작하고, 재화로 반복 구매할 수 있다. 구매 한 번은 구매 전 레벨 기준의 다음 1레벨 비용을 차감한 뒤 해당 업그레이드의 레벨을 1 올린다. 각 업그레이드는 레벨을 가지며, 비용은 레벨마다 증가한다. 아래 표의 행 순서는 상점 표시 순서와 동일하다.

| 이름 | 기본 비용 | 성장률 | 효과 | 의도 |
| --- | ---: | ---: | --- | --- |
| 낡은 몽둥이 | 3 | 1.45 | 클릭 피해량 = 1 + 레벨 | 가장 직관적인 기본 강화 |
| 삐걱대는 투석기 | 12 | 1.55 | 레벨 1 이상일 때 5초마다 현재 클릭 피해량 * (1 + 레벨) 자동 피해 | 클릭 피로 완화 |
| 고블린 미끼 주머니 | 20 | 1.6 | 처치 보상 = 기본 처치 보상 + 레벨 | 재화 성장 선택지 |
| 진흙 함정 | 90 | 1.85 | 새 고블린 등장 시 준비 레벨 고정, 첫 직접 공격 배율 = 1 + 2 * 준비 레벨 | 첫 직접 타격 쾌감 |

### 업그레이드 설계 원칙

- `낡은 몽둥이`는 항상 유효한 기본 선택지다.
- `고블린 미끼 주머니`는 보상 성장을 담당한다.
- `진흙 함정`은 새 고블린 등장 시점의 레벨을 `mudTrapArmedLevel`로 고정하고, 첫 번째 직접 공격이 적중하면 `0`으로 소모된다.
- 전투 중 `진흙 함정`을 새로 구매하거나 추가 구매해도 현재 고블린의 `mudTrapArmedLevel`은 바뀌지 않는다.
- `삐걱대는 투석기`는 클릭 피로 완화와 처치 직전 마무리를 돕되, 직접 클릭을 대체하지 않는다.
- `삐걱대는 투석기` 레벨이 0이면 쿨다운과 HUD 상태 표시가 비활성이다.
- 전투 중 `삐걱대는 투석기`를 처음 구매하면 `catapultCooldownRemainingMs`를 `5000`으로 설정하고 그때부터 카운트다운을 시작한다.
- 이미 `삐걱대는 투석기`를 보유한 상태에서 추가 구매하면 쿨다운은 리셋하지 않고, 다음 자동 공격 피해량만 새 투석기 레벨 기준으로 갱신한다.
- `x10 구매`, 최대 구매, 구매 배수 선택은 MVP에 포함하지 않는다.
- MVP에서는 업그레이드 되팔기와 초기화 기능을 넣지 않는다.

## 8. 게임 상태 모델

저장 가능한 상태는 렌더링 객체와 분리한다.

```ts
type GameState = {
  goblinHp: number;
  mudTrapArmedLevel: number;
  catapultCooldownRemainingMs: number;
  defeatedCount: number;
  coins: number;
  upgrades: Record<UpgradeId, number>;
};

type SaveData = {
  saveVersion: 1;
  state: GameState;
  savedAt?: number;
};
```

현재 고블린 레벨은 `currentGoblinLevel = defeatedCount + 1`로 계산한다. 현재 고블린 최대 체력은 `currentGoblinMaxHp = calculateGoblinMaxHp(currentGoblinLevel)`로 계산한다. `GameState`에는 `goblinLevel`과 `goblinMaxHp`를 저장하지 않는다.

`GameState.goblinHp`는 런타임에서는 항상 `0` 이상 `currentGoblinMaxHp` 이하로 유지한다. 피해 적용 후에는 `max(0, goblinHp - damage)`로 저장하며, 오버킬 피해가 발생해도 음수 체력을 저장하지 않는다. 단, `goblinHp === 0`은 처치 처리로 이어지는 일시 상태이며 현재 고블린의 저장 가능한 복원 상태가 아니다.

`mudTrapArmedLevel`과 `catapultCooldownRemainingMs`는 계산 값이 아니라 현재 고블린 전투의 진행 상태이므로 저장한다. `mudTrapArmedLevel`은 `0`이면 미준비 또는 소모됨, `1` 이상이면 현재 고블린에게 고정된 진흙 함정 준비 레벨을 뜻한다. 투석기 레벨이 0이면 `catapultCooldownRemainingMs`는 `0`으로 두고, 전투 계산과 HUD 표시에서 사용하지 않는다.

투석기 쿨다운 계산에 쓰는 `performance.now()` 값 자체나 마지막 프레임 기준 시간은 `GameState`와 `SaveData`에 저장하지 않는다. 저장 가능한 값은 현재 전투 상태인 `catapultCooldownRemainingMs`뿐이다.

`GameState.upgrades`는 MVP 4종 업그레이드 키만 가진다. 각 업그레이드 레벨은 `Number.isSafeInteger(value) && value >= 0`을 만족하는 유한한 0 이상의 안전한 정수여야 한다.

`낡은 몽둥이`의 첫 구매 `추천` 표시는 저장하지 않는다. 이 표시는 `낡은 몽둥이` 레벨이 0인지에서 계산되는 파생 UI 상태이며, `GameState`와 `SaveData`에 별도 추천 플래그를 두지 않는다. 모바일 접힌 `상점` 버튼의 구매 가능 표시와 구매 가능 개수도 저장하지 않는다. 구매 가능 개수는 현재 `coins`와 각 업그레이드의 다음 1레벨 비용에서 파생하며, 추천 상태는 이 개수에 포함하지 않는다.

저장 복원 시 모든 숫자 상태는 필드별 규칙에 따라 `Number.isSafeInteger(value)` 기준으로 검증한다. `defeatedCount`와 `coins`는 유효한 0 이상의 안전한 정수일 때만 사용하고, 아니면 0으로 정규화한다. `goblinHp`, `catapultCooldownRemainingMs`, `mudTrapArmedLevel`, 업그레이드 레벨은 각 필드의 의미 범위에 맞춰 별도로 정규화한다.

`saveVersion`은 저장 포맷 호환성 판단 기준이며, MVP 초기 저장 버전은 `1`로 고정한다. 게임 진행 상태가 아니므로 `GameState`가 아니라 `SaveData`에 둔다.

`saveVersion`과 `savedAt`은 게임 상태가 아니라 저장 메타데이터다. `savedAt`은 저장 시각 표시나 디버깅에만 사용하며, 체력, 재화, 투석기 쿨다운, 처치 보상 계산에는 사용하지 않는다. 저장 시각 표시가 필요하지 않다면 MVP에서 생략할 수 있다. `Date.now()`나 시스템 시각도 MVP 진행 계산에는 사용하지 않는다.

계산 값은 저장하지 않고 필요할 때 다시 계산한다.

- 클릭 피해량
- 현재 고블린 레벨
- 현재 고블린 최대 체력
- 진흙 함정 첫 직접 공격 배율
- 다음 자동 공격 피해량
- 처치 보상
- 다음 업그레이드 비용

체력 바 잔상 값은 `GameState`에 저장하지 않는 순수 렌더링 상태다. 처치 판정, 저장, 다음 고블린 체력 계산, 테스트의 실제 체력 계산에는 `goblinHp`와 `defeatedCount`에서 파생한 `currentGoblinMaxHp`만 사용한다.

## 9. 시스템 구성

### 핵심 모듈

- `gameState`: 현재 저장 가능한 게임 상태를 관리한다.
- `progression`: 고블린 체력, 보상, 업그레이드 비용을 계산한다.
- `combat`: 클릭 피해, 투석기 쿨다운, 다음 자동 공격 피해, 처치 판정을 처리한다.
- `upgrades`: 구매 가능 여부와 구매 적용을 처리한다.
- `save`: 저장, 불러오기, 저장 실패 처리, 저장 초기화, 자동 저장 보조 트리거를 처리한다.
- `GoblinScene`: Phaser 장면, 스프라이트, 애니메이션, 입력 이벤트를 담당한다.
- `Hud`: DOM 기반 숫자 표시와 상점 UI를 담당한다.

### 책임 경계

- 게임 규칙은 TypeScript 순수 함수에 둔다.
- Phaser 장면은 입력을 받아 규칙 모듈에 전달하고, 결과를 시각적으로 표현한다.
- DOM HUD는 상태를 표시하고 업그레이드 구매 명령을 보낸다.
- 저장 시스템은 `GameState`와 선택 저장 메타데이터를 `SaveData` 형태로 저장한다. 게임 규칙은 `GameState`만 읽고, 저장 메타데이터를 진행 계산에 사용하지 않는다.

## 10. 입력

| 액션 | 마우스 | 터치 | 키보드 |
| --- | --- | --- | --- |
| 고블린 공격 | 직접 공격 영역 안에서 시작한 `pointerdown` | 직접 공격 영역 안에서 시작한 `pointerdown` | 게임 루트 또는 전투 화면 포커스 상태의 Space `keydown` |
| 업그레이드 구매 | 버튼 클릭 | 버튼 탭 | 구매 버튼 포커스 상태의 Enter 또는 Space |
| 상점 열기/닫기 | 버튼 클릭 | 하단 `상점` 버튼 또는 시트 내부 `닫기` 버튼 탭 | S. Escape는 상점 닫기에 사용하지 않음 |
| 저장 초기화 | 설정 버튼 + 확인 모달 | 설정 버튼 + 확인 모달 | 확인 모달에서 Escape는 취소 |
| 로드 오류 처리 | 차단 모달의 `다시 시도`, `새 게임 시작`, `취소` 버튼 클릭 | 차단 모달의 `다시 시도`, `새 게임 시작`, `취소` 버튼 탭 | 모달 버튼 포커스 상태의 Enter 또는 Space. Escape는 무시 |

키보드 입력은 접근성 보조용이며, MVP의 핵심 입력은 클릭과 터치다.

MVP에서 Escape는 모바일이나 데스크톱 상점 닫기 단축키로 사용하지 않는다. 상점은 명시적 버튼과 `S` 키로만 닫는다. Escape는 닫아도 유효한 이전 상태로 돌아갈 수 있는 취소 가능 모달에서만 `취소`와 동일하게 동작한다. 저장 초기화 확인 모달에서는 Escape를 누르면 취소 버튼과 동일하게 아무 상태도 변경하지 않는다. 로드 오류 차단 모달처럼 현재 게임을 시작할 수 없고 사용자가 `다시 시도` 또는 `새 게임 시작`을 선택해야 하는 상태에서는 Escape가 모달을 닫거나 암묵적으로 `취소`를 실행하지 않는다. 이 모달의 `취소` 버튼은 삭제나 새 게임 시작을 하지 않고 로드 오류 상태에 머무는 명시 선택지이며, 정상 플레이 화면으로 돌아가는 닫기 동작이 아니다.

고블린 공격 액션의 마우스 `pointerdown`, 터치 `pointerdown`, Space `keydown`은 모두 같은 직접 공격 큐로 처리한다. 포인터 입력은 고블린 중심의 보이지 않는 고정 직접 공격 영역 안에서 시작한 `pointerdown`만 공격 후보로 처리한다. 같은 포인터 조작에서 뒤따르는 `click` 이벤트는 고블린 공격으로 처리하지 않으며, 직접 공격 영역 밖에서 시작한 포인터가 영역 안으로 들어와도 공격하지 않는다. Space는 게임 루트 또는 전투 화면에 포커스가 있을 때 발생한 `keydown`만 직접 공격 후보로 처리한다. Space `keyup`은 공격으로 처리하지 않고, `event.repeat`이 `true`인 Space `keydown`은 무시한다. 전투 포커스 상태에서 Space를 직접 공격 후보로 처리할 때만 `preventDefault()`로 페이지 스크롤을 막는다. 멀티터치는 같은 80ms 제한과 직접 공격 큐 규칙으로 중복 공격을 막으며, 길게 누르기나 Space 키 홀드에 의한 자동 반복 공격은 지원하지 않는다.

전투 영역 컨테이너는 키보드 포커스를 받을 수 있는 요소여야 한다. 직접 공격 영역 안에서 `pointerdown`이 성공적으로 직접 공격 후보가 되면 전투 영역 컨테이너에 포커스를 준다. 상점, 버튼, 모달 위 입력은 전투 영역으로 포커스를 훔치지 않는다. 모바일 상점을 열면 시트 컨테이너 또는 시트 제목으로 포커스를 옮기고, 이 포커스 상태의 Enter 또는 Space는 구매나 닫기 입력으로 처리하지 않는다. 모바일 상점 시트를 시트 내부 `닫기` 버튼, 하단 `상점` 토글 버튼, 키보드 `S` 중 어느 경로로 닫아도 같은 닫기 동작을 수행하고, 포커스는 하단 `상점` 버튼으로 돌린다. 저장 초기화 확인 모달이나 로드 오류 모달이 닫히는 경우에는 모달을 연 버튼 또는 모달 진입 전 포커스 요소로 포커스를 돌린다. 로드 오류 상태처럼 게임이 아직 시작되지 않은 상태에서는 전투 영역에 포커스를 주지 않는다. 모달 내부를 제외하고 Tab 키 이동 자체를 가로막는 포커스 트랩은 만들지 않는다.

상점, 버튼, 확인 모달 위에서 발생한 `pointerdown`, 터치, 클릭은 해당 UI에만 전달하고 고블린 공격으로 처리하지 않는다. UI 입력은 전투 입력보다 우선한다. 상점 버튼, 구매 버튼, 닫기 버튼, 확인 모달 버튼처럼 포커스 가능한 UI에 포커스가 있으면 Space는 해당 UI의 기본 동작 또는 UI 입력으로 처리하고 고블린 공격으로 처리하지 않는다. Enter는 포커스된 UI 버튼 활성화에 사용하며 고블린 공격으로 사용하지 않는다. 비용 부족 업그레이드 버튼은 포커스 가능한 `구매 불가` 상태이며, 평상시에는 클릭, 탭, Space, Enter 입력을 받으면 구매 실패 피드백만 표시한다. 300ms 처치 전환 중에는 구매 버튼 포커스를 가능하면 유지하되, 구매 핸들러가 즉시 반환해 구매 입력을 처리하지 않고 큐에 쌓지 않으며 비용 부족 실패 피드백도 표시하지 않는다. 전환 종료 후 이전 Enter, Space, 클릭, 탭 입력을 자동 재실행하지 않는다. 상점 시트 밖에 남아 있는 직접 공격 영역을 누르면 직접 공격은 계속 가능하며, 시트 바깥 탭으로 상점을 닫지 않는다. 저장 초기화 확인 모달이나 로드 오류 차단 모달처럼 배경 플레이를 막아야 하는 모달이 열린 경우에는 배경 입력을 차단하고, Space도 고블린 공격으로 처리하지 않는다. 로드 오류 상태에서는 새 게임이 시작되기 전까지 고블린 공격과 업그레이드 구매를 처리하지 않는다. 저장소 접근 실패로 인한 비저장 세션에서는 저장 초기화 버튼을 실제 비활성화하거나, 활성 상태로 남기더라도 입력 시 상태를 바꾸지 않고 짧은 경고만 표시한다.

- 고블린 피격 프레임 1종
- 고블린 처치 프레임 또는 사라짐 애니메이션 1종
- 클릭 타격 이펙트 1종
- 처치 파티클 1종
- 재화 아이콘 1종
- 업그레이드 아이콘 4종
- 배경 이미지 1종
- 버튼, 패널, 체력 바 UI 스타일

그래픽 스타일은 모바일 가독성을 우선한 2D 카툰으로 통일한다. 고해상도 일러스트풍이나 과도하게 디테일한 페인터리 스타일은 MVP에 사용하지 않는다. 고블린은 굵은 실루엣, 단순한 명암, 큰 눈과 입 표정, 명확한 피격 포즈를 가져야 한다.

UI 아이콘은 같은 카툰 톤을 따르되, 작은 크기에서도 읽히도록 단순한 형태와 제한된 색 수를 사용한다. MVP에서는 고블린 종류를 늘리지 않고 색상 변화나 작은 장식으로 레벨 상승감을 표현한다.

고블린의 기본 클릭 영역, 영구 스케일, 기본 실루엣은 레벨과 무관하게 고정한다. 5레벨 단위로 색조, 눈빛, 작은 투구, 귀걸이, 흉터 같은 소품을 바꿔 진행감을 준다.

### 사운드 선택 구현

- 클릭 타격음
- 고블린 처치음
- 재화 획득음
- 업그레이드 구매음
- 구매 실패음
- 짧은 배경 음악 1곡

사운드는 MVP 필수 완료 기준에 포함하지 않는다. 사운드를 구현하는 경우에는 전역 음소거 토글과 음소거 상태 저장을 함께 구현한다. 모바일 자동재생 정책 대응, 볼륨 UI, 음소거 상태 저장은 사운드를 필수로 둘 때 테스트 범위를 키우므로 MVP 필수 범위에서 제외한다.

## 12. 저장과 복구

### 자동 저장

MVP의 `localStorage` 저장 키는 `goblin-clicker.save` 하나로 고정한다. 저장 포맷 버전은 키 이름이 아니라 `SaveData.saveVersion`으로 판단한다. MVP에서는 다중 슬롯, 버전별 키, 백업 키를 사용하지 않는다.

- 고블린 처치 시 저장한다.
- 업그레이드 구매 성공 시 저장한다.
- 업그레이드 구매 실패 시에는 저장하지 않는다.
- 진흙 함정이 소모될 때 저장한다. 단, 같은 입력에서 처치까지 발생하면 처치 처리 후 최종 상태 저장 1회로 병합한다.
- 10초마다 자동 저장을 시도한다.

같은 업데이트 흐름에서 여러 저장 사유가 발생하면 저장 트리거를 병합하고, 중간 상태가 아니라 모든 처리가 끝난 뒤 최종 `SaveData`만 1회 저장한다. 저장 사유가 직접 공격, 투석기 자동 공격, 10초 자동 저장 타이머 중 무엇인지는 병합 규칙에 영향을 주지 않는다. 예를 들어 직접 공격으로 `진흙 함정`만 소모되고 고블린이 살아 있으면 현재 전투 상태를 저장하지만, 같은 공격으로 고블린까지 처치했다면 확정된 다음 고블린 상태를 저장한다. 투석기 자동 공격 처치와 10초 자동 저장 타이머가 같은 업데이트 흐름에 겹쳐도 처치 수, 재화, 다음 고블린 상태, 투석기 쿨다운까지 확정한 최종 상태만 저장한다.

업그레이드 구매는 입력 1회당 독립 트랜잭션으로 처리한다. 빠른 연속 구매 입력이 들어와도 각 입력은 직전 구매 결과가 반영된 최신 메모리 상태에서 순차 평가하며, 성공한 구매마다 해당 입력 처리 후 최종 상태 기준으로 저장 트리거를 발생시킨다. 비용 부족 실패나 실제 입력 차단 상태의 구매 입력은 저장 트리거를 발생시키지 않는다.

10초 자동 저장은 전투 규칙에 쓰이는 정밀 타이머가 아니라 저장 보조 트리거다. 전투 처리 중간 상태를 캡처하는 시스템이 아니라, 현재 확정 상태를 보조로 저장하는 기능이다. 구현은 단순 브라우저 타이머를 사용할 수 있으며, 투석기 쿨다운처럼 `performance.now()` 기반 전투 타이머로 취급하지 않는다. 브라우저 타이머가 지연되거나 `document.visibilityState === "hidden"` 상태에서 재개되어도 밀린 자동 저장을 여러 번 몰아서 실행하지 않고, 다음 자동 저장 시도 1회만 수행한다. 로드 오류 상태와 저장소 접근 실패로 인한 비저장 세션에서는 자동 저장을 실행하지 않으며, 게임이 시작된 세션에서 저장할 `SaveData`가 있고 저장소 접근 실패 상태가 아닐 때만 자동 저장을 시도한다.

앱 시작 시 `localStorage` 읽기 자체가 실패한 경우에는 저장 데이터 없음과 구분한다. 읽기 실패는 기존 저장이 없다는 뜻이 아니라 저장소 접근 실패이므로, 게임은 시작 상태로 열 수 있지만 이 세션을 저장소 접근 실패로 인한 비저장 세션으로 표시한다. HUD에는 `저장소에 접근할 수 없어 저장되지 않음` 같은 명확한 경고와 새로고침하면 진행이 사라질 수 있음을 표시한다. 이 상태에서도 고블린 공격, 처치, 재화 획득, 업그레이드 구매는 메모리 상태에서 정상 처리한다. 다만 `goblin-clicker.save`에 자동으로 시작 상태를 저장하지 않고, 10초 자동 저장, 처치 저장, 구매 저장 같은 저장 트리거도 실제 쓰기를 시도하지 않으며 저장 실패 상태로 취급한다. MVP에서는 저장소 접근 실패 복구 UI를 만들지 않으며, 사용자는 브라우저 설정이나 권한 문제를 해결한 뒤 새로고침해 다시 로드할 수 있다.

`localStorage` 쓰기가 실패해도 현재 세션 플레이는 계속 가능해야 한다. 마지막 저장 시도가 실패하면 HUD에 `저장되지 않음` 같은 작고 명확한 경고를 표시하고, 저장 불가 상태에서 새로고침하면 진행이 사라질 수 있음을 알린다. 이후 어떤 저장 트리거든 `goblin-clicker.save` 쓰기에 성공하면 즉시 이 경고를 해제한다. 별도 복구 루프를 만들지 않고 다음 저장 트리거에서 다시 저장을 시도한다.

`새 게임 시작` 또는 `저장 초기화`에서 `goblin-clicker.save` 삭제가 성공한 뒤 시작 상태 저장이 실패해도 새 게임 세션은 계속 시작한다. 사용자가 파괴적 동작을 확정했고 기존 저장 삭제가 성공했다면 이전 저장을 복구하려고 하지 않는다. 이 상황은 기존 `localStorage` 쓰기 실패와 동일한 저장 실패 상태로 다루며, HUD에 `저장되지 않음` 경고를 표시하고 다음 저장 트리거에서 다시 저장을 시도한다.

`새 게임 시작` 또는 `저장 초기화`에서 `goblin-clicker.save` 삭제 자체가 실패하면 새 게임을 시작하지 않는다. 기존 저장은 유지된 것으로 간주하고 시작 상태로 재초기화하지 않는다. 사용자에게 `저장 데이터를 삭제할 수 없습니다.` 수준의 명확한 경고를 표시하며, 삭제를 다시 시도하거나 취소할 수 있게 한다. 삭제 실패 상태에서 저장하지 않는 임시 새 게임을 시작하지 않는다.

처치 후 300ms 전환은 화면 연출 상태이므로 저장하지 않는다. 전환 타이머, 남은 전환 시간, 전환 중 입력 차단 상태는 `GameState`나 `SaveData`에 포함하지 않는다. 처치 저장은 `defeatedCount`, 재화, 다음 고블린의 `goblinHp`, `mudTrapArmedLevel`, `catapultCooldownRemainingMs`가 확정된 뒤 발생한다.

정상 저장 플로우에서 저장되는 현재 고블린의 유효 체력 범위는 `1..currentGoblinMaxHp`다. 저장 데이터에 `goblinHp === 0`이 있으면 처치 처리 중간 상태가 남은 비정상 저장으로 보고, 불러오기 시 현재 고블린을 0 체력으로 복원하지 않는다.

### 불러오기

- `localStorage` 읽기에 성공했지만 저장 데이터가 없으면 시작 상태의 `GameState`와 `SaveData`를 생성하고, 생성 직후 `goblin-clicker.save`에 즉시 저장을 시도한다.
- 첫 실행 시작 상태 저장에 성공하면 경고 없이 게임을 시작하고, 실패하면 게임은 시작하되 HUD에 `저장되지 않음` 경고를 표시한다.
- 앱 시작 시 `localStorage` 읽기 자체가 실패하면 시작 상태로 게임을 열 수 있지만, 저장소 접근 실패로 인한 비저장 세션으로 표시한다.
- 저장소 접근 실패로 인한 비저장 세션에서는 자동으로 시작 상태를 저장하지 않고, 이후 저장 트리거도 실제 쓰기를 시도하지 않는다.
- 저장소 접근 실패로 인한 비저장 세션은 저장소에는 접근했지만 데이터 포맷을 복원할 수 없는 파싱 실패나 마이그레이션 실패와 구분한다.
- 저장 데이터가 있으면 가장 먼저 `SaveData.saveVersion`을 확인한다.
- `saveVersion: 1`인 경우에만 필드별 정규화와 복원을 진행한다.
- `saveVersion`이 누락된 경우에는 레거시 저장 데이터로 간주하고 마이그레이션을 시도한다.
- MVP에서 명시적으로 지원하는 레거시 포맷은 없으므로, `saveVersion` 누락 데이터의 마이그레이션은 실패로 처리한다.
- 알 수 없는 포맷을 현재 포맷처럼 추론해 부분 복원하지 않는다.
- 오프라인 보상은 제공하지 않으며, `mudTrapArmedLevel`과 `catapultCooldownRemainingMs`는 저장 당시 값을 기준으로 복원한다. 저장 후 불러오기까지 경과한 시간과 `document.visibilityState === "hidden"` 동안의 경과 시간은 투석기 쿨다운에 반영하지 않는다. 저장 복원 시 `performance.now()` 값은 복원하지 않고, 복원 이후의 새 세션 기준 시간으로 다음 delta를 계산한다.
- `defeatedCount`는 `Number.isSafeInteger(value) && value >= 0`이면 사용하고, 아니면 0으로 정규화한다.
- `coins`는 `Number.isSafeInteger(value) && value >= 0`이면 사용하고, 아니면 0으로 정규화한다.
- 저장 데이터를 불러올 때 `currentGoblinLevel = defeatedCount + 1`과 `currentGoblinMaxHp = calculateGoblinMaxHp(currentGoblinLevel)`를 계산한다.
- 저장된 `goblinHp`가 유효한 안전한 정수이고 `1..currentGoblinMaxHp` 범위이면 그대로 사용한다.
- 저장된 `goblinHp`가 0이면 비정상 전투 중간 상태로 보고 `currentGoblinMaxHp`로 정규화한다.
- 누락, `NaN`, `Infinity`, 음수, 소수, 숫자가 아닌 값이면 `currentGoblinMaxHp`로 정규화해 현재 고블린을 새 체력으로 복원한다.
- 저장 데이터를 불러올 때 `upgrades`는 MVP 4종 업그레이드 키만 신뢰한다.
- MVP 4종 업그레이드 키가 누락되어 있으면 해당 레벨은 0으로 채운다.
- 알 수 없는 업그레이드 키는 무시하고 `GameState.upgrades`에 보존하지 않는다.
- 각 업그레이드 레벨은 `Number.isSafeInteger(value) && value >= 0`일 때만 유효하다. 음수, 소수, `NaN`, `Infinity`, 숫자가 아닌 값은 0으로 정규화한다.
- `catapultCooldownRemainingMs`는 저장 시 정수 ms로 저장한다. 투석기 레벨이 0이면 항상 0으로 정규화한다. 투석기 레벨이 1 이상이면 유효한 0 이상의 안전한 정수일 때 `0..5000` 범위로 클램프하고, 유효하지 않으면 `5000`으로 정규화한다.
- `mudTrapArmedLevel`은 유효한 0 이상의 안전한 정수일 때 `0..현재 진흙 함정 레벨` 범위로 클램프한다. 유효하지 않으면 0으로 정규화한다.
- 필드별 정규화로 복원 가능한 저장 데이터는 사용자에게 별도 경고를 띄우지 않고 조용히 보정한 뒤 게임을 계속 시작한다.
- 개발 모드에서는 콘솔 로그나 디버그 표시로 정규화 사실을 남길 수 있다.
- 처치 후 300ms 전환 중 새로고침해도 전환 연출은 복원하지 않고, 저장된 다음 고블린 상태를 즉시 표시한다.
- `savedAt`이 있더라도 체력, 재화, 투석기 쿨다운, 처치 보상 계산에 사용하지 않는다. `Date.now()`나 시스템 시각도 진행 계산에 사용하지 않는다.
- 지원하지 않는 `SaveData.saveVersion`이면 마이그레이션을 시도한다.
- 저장 데이터 파싱 실패, `SaveData.saveVersion` 누락 데이터의 마이그레이션 실패, 지원하지 않는 버전의 마이그레이션 실패는 로드 오류 상태로 전환하고 차단 모달을 표시한다.
- 로드 오류 모달 문구는 `저장 데이터를 불러올 수 없습니다. 새 게임으로 시작하면 기존 저장은 삭제됩니다.`처럼 새 게임 시작 시 기존 저장이 삭제된다는 결과를 명확히 알린다.
- 로드 오류 모달에서 `새 게임 시작`을 누르면 `goblin-clicker.save` 값을 삭제하고 시작 상태로 새 게임을 시작한 뒤 시작 상태를 저장한다.
- `새 게임 시작`에서 `goblin-clicker.save` 삭제는 성공했지만 시작 상태 저장이 실패하면 새 게임 세션을 계속 시작하고 저장 실패 상태를 표시한다.
- `새 게임 시작`에서 `goblin-clicker.save` 삭제가 실패하면 기존 저장을 유지한 것으로 간주하고 새 게임을 시작하지 않으며, `저장 데이터를 삭제할 수 없습니다.` 경고와 다시 시도 또는 취소 선택지를 유지한다.
- 로드 오류 모달에서 `취소`를 누르면 기존 저장 데이터를 삭제하지 않고, 새 게임도 시작하지 않는다.
- 취소 후에는 로드 오류 상태에 머물며 다시 시도하거나 새 게임 시작을 선택할 수 있게 한다.
- 로드 오류 차단 모달에서는 Escape를 눌러도 모달을 닫거나 암묵적으로 `취소`를 실행하지 않는다.
- 로드 오류 상태에서 `다시 시도`를 누르면 현재 `localStorage` 값을 다시 읽고, 파싱, `SaveData.saveVersion` 확인, 마이그레이션, 필드별 정규화 절차를 처음부터 다시 실행한다.
- 다시 시도가 성공하면 로드 오류 모달을 닫고 복원된 게임을 시작한다.
- 다시 시도가 실패하면 기존 저장 데이터를 삭제하지 않고 같은 로드 오류 상태와 차단 모달을 유지한다.
- MVP에서는 파일 선택, 백업 복구, 수동 JSON 편집, 저장하지 않고 임시 플레이, 백업 다운로드, 수동 복구 UI를 제공하지 않는다.

### 저장 초기화

저장 초기화는 바로 실행하지 않고 단순 확인 모달을 거친다. 모달 문구는 `정말 새 게임으로 시작할까요?` 수준으로 짧게 유지한다.

- 확인 버튼을 누르면 `localStorage`의 `goblin-clicker.save` 값을 삭제하고 시작 상태로 재초기화한 뒤 시작 상태를 저장한다.
- `goblin-clicker.save` 삭제는 성공했지만 시작 상태 저장이 실패하면 새 게임 세션을 계속 진행하고 저장 실패 상태를 표시한다.
- `goblin-clicker.save` 삭제가 실패하면 기존 저장을 유지한 것으로 간주하고 시작 상태로 재초기화하지 않으며, `저장 데이터를 삭제할 수 없습니다.` 경고와 다시 시도 또는 취소 선택지를 유지한다.
- 취소 버튼을 누르거나 Escape를 누르면 아무 상태도 변경하지 않는다.
- 저장소 접근 실패로 인한 비저장 세션에서는 저장 초기화를 실행하지 못하게 한다.
- 비저장 세션에서 저장 초기화 버튼은 실제 비활성화하거나, 누르더라도 `저장소에 접근할 수 없어 초기화할 수 없습니다` 같은 짧고 명확한 경고만 표시하고 상태를 바꾸지 않는다.
- 비저장 세션에서 현재 세션만 로컬로 초기화하는 기능은 MVP에 넣지 않는다.
- 저장소 접근 실패 상태에서는 저장, 삭제, 초기화처럼 `goblin-clicker.save`에 쓰거나 지우는 동작을 수행하지 않는다.
- 백업, 되돌리기, 파일 내보내기, 복구 UI는 MVP에 포함하지 않는다.

MVP에서는 오프라인 보상은 제공하지 않는다. 오프라인 보상은 방치형 밸런스에 큰 영향을 주므로 별도 기능으로 다룬다.

사운드를 구현한 경우 음소거 상태는 저장하고 새로고침 후 복원한다. 사운드를 구현하지 않아도 MVP 완료 기준을 통과할 수 있다.

## 13. 밸런스 목표

- 첫 처치까지 필요한 클릭 수: 5회
- 첫 업그레이드 구매 가능 시점: 2번째 고블린 처치 직후
- 첫 두 고블린 처치 후 보유 재화: 3
- 첫 구매 가능 업그레이드: `낡은 몽둥이`. 첫 비용 3으로 맞춰 직접 클릭 피해 증가를 가장 먼저 학습시킨다.
- 첫 업그레이드 구매까지 필요한 직접 공격 수: 10~12회
- 5분 플레이 후 기대 처치 수: 25~40마리. 이 수치는 구현 후 밸런스 튜닝에서 검증한다.
- 5분 플레이 후 보유 업그레이드 종류: 2~3종
- 5분 안에는 `낡은 몽둥이`, `삐걱대는 투석기`, `고블린 미끼 주머니` 중 2종 이상 구매를 목표로 한다.
- `진흙 함정`은 5분 이후 첫 중기 목표로 둔다.
- MVP 밸런스 검증 범위: 첫 10분 또는 대략 40마리 처치까지
- 첫 10분 동안 가장 빠른 진행 방법: 직접 클릭
- 자동 피해만으로 같은 레벨대 고블린을 안정적으로 처치하는 상태: MVP 목표 밖

## 14. 피드백과 연출

- 클릭할 때 고블린이 짧게 흔들리거나 눌린다.
- 고블린의 표정 변화, 눌림, 피격 반응은 모바일 화면에서도 즉시 읽혀야 한다.
- 피격 순간의 squash/stretch와 짧은 흔들림은 허용하지만, 레벨별 영구 크기 변화로 사용하지 않는다.
- 피격 애니메이션, 장식, squash/stretch는 보이는 스프라이트만 바꾸며 직접 공격 판정 영역을 움직이거나 줄이지 않는다.
- 피해 숫자가 고블린 위로 떠오른다.
- 피해 숫자는 실제 가한 총 피해량을 표시한다. 오버킬이 발생해도 `GameState.goblinHp`, 체력 숫자, 즉시 체력 바는 최소 0으로 표시하며, 초과 피해는 다음 고블린 체력에 영향을 주지 않는다.
- 직접 공격과 투석기 자동 공격은 각각 별도 피해 숫자를 표시한다. 같은 순간에 직접 공격과 투석기 피해가 겹쳐도 합산하지 않고 별도 숫자로 표시한다.
- 피해 숫자의 최대 동시 표시 개수는 `6개`로 제한한다. 6개를 초과하면 가장 오래된 피해 숫자부터 제거한다.
- 피해 숫자는 약 `600ms` 동안 위로 떠오르며 사라진다.
- 투석기 자동 공격은 직접 공격과 구분되는 자동 타격 연출과 피해 숫자를 사용한다. 투석기 피해 숫자는 작은 아이콘, 색, 또는 출현 위치 차이로 직접 공격 피해 숫자와 구분한다.
- 실제 고블린 체력 숫자와 즉시 체력 바는 피해 적용 즉시 새 값으로 갱신한다.
- 체력 숫자와 즉시 체력 바는 최소 0으로 표시한다.
- 체력 바 잔상 효과는 실제 체력값과 분리된 순수 렌더링 연출이다. 잔상 바만 약 `250ms` 늦게 따라온다.
- 잔상 바 값은 처치 판정, 저장, 다음 고블린 체력 계산, 테스트의 실제 체력 계산에 사용하지 않는다.
- 고블린이 처치되면 실제 체력과 즉시 체력 바는 0 또는 처치 상태가 되고, 잔상 바는 처치 전환 연출 안에서만 따라온다.
- 다음 고블린이 표시될 때 실제 체력 바와 잔상 바 모두 새 고블린의 최대 체력 상태로 재설정한다.
- 처치 시 고블린이 작게 터지거나 먼지로 사라지고, 다음 고블린은 300ms 처치 전환 시간 뒤에 표시한다.
- 300ms 처치 전환 중 렌더러와 HUD는 처치 연출 스냅샷을 보여준다. 전역 상태 표시는 즉시 갱신하되, 고블린 레벨, 체력 바, 현재 체력 숫자는 다음 고블린이 표시될 때 새 값으로 갱신한다.
- 오버킬이 발생하면 처치 연출을 조금 더 강하게 보여줄 수 있다. 이 연출은 보상, 처치 수, 다음 고블린 체력에 영향을 주지 않는다.
- 무음 상태에서도 피해 숫자, 체력 바, 고블린 피격 반응, 처치 파티클만으로 전투 결과가 읽혀야 한다.
- 업그레이드 구매 가능 상태는 버튼 강조로 보여준다.
- 구매 불가 상태는 비용 부족을 명확히 표시하고, 색상만이 아니라 텍스트로 남은 비용을 보여준다.
- 비용 부족으로 업그레이드 구매가 실패하면 해당 버튼에만 짧은 오류 피드백을 준다. 사운드를 구현한 경우 구매 실패음은 선택적으로 재생할 수 있지만 MVP 필수 조건은 아니다.
- 업그레이드 구매 성공 시 버튼 눌림, 레벨 숫자 증가 강조, 재화 숫자 갱신, 다음 효과와 다음 비용 갱신 같은 짧은 로컬 피드백만 사용한다.
- 전환 중 구매 버튼의 입력 차단 이유는 버튼 단위 상태 텍스트로만 표시한다. 문구는 `잠시 후 구매 가능` 또는 `처치 처리 중`처럼 짧게 유지하며, 전역 토스트, 모달, 화면 전체 알림은 사용하지 않는다.
- 비용 부족 상태와 전환 상태가 겹치면 전환 상태가 우선이다. 전환 중에는 `녹슨 동전 부족`보다 `잠시 후 구매 가능` 같은 전환 상태 문구를 표시하고, 전환이 끝나면 다시 구매 가능 또는 비용 부족 상태 텍스트로 돌아간다.
- 저장 실패 상태는 HUD에 `저장되지 않음` 같은 짧은 경고로 표시한다. 이 경고는 일반 세션에서는 마지막 저장 시도 실패를 뜻하며, 이후 `goblin-clicker.save` 쓰기에 성공하면 즉시 해제한다.
- 저장소 접근 실패로 인한 비저장 세션에서는 HUD에 `저장소에 접근할 수 없어 저장되지 않음` 같은 더 구체적인 경고를 표시하고, 새로고침하면 비저장 세션의 진행이 사라질 수 있음을 알린다.
- 비저장 세션에서 저장 초기화를 누른 경우에는 `저장소에 접근할 수 없어 초기화할 수 없습니다` 같은 짧고 명확한 경고를 표시하고, 전역 토스트 큐나 긴 복구 안내는 만들지 않는다.
- 로드 오류, 저장 데이터 삭제 실패, 파싱 실패, 마이그레이션 실패 같은 차단 오류는 HUD의 `저장되지 않음` 경고가 아니라 모달 또는 명확한 오류 문구로 분리한다.

## 15. 접근성과 반응형

- 모든 상점 버튼은 키보드 포커스를 받을 수 있어야 한다.
- 전투 영역 컨테이너는 키보드 포커스를 받을 수 있어야 한다.
- 비용 부족 업그레이드 버튼도 실제 `disabled`가 아니므로 키보드 포커스를 받을 수 있어야 한다.
- 300ms 처치 전환 중 구매 버튼은 가능하면 실제 HTML `disabled` 대신 `aria-disabled` 또는 동등한 상태 표시와 구매 핸들러 차단을 사용해 키보드 포커스를 유지한다.
- 전환 중 구매 버튼의 상태 문구는 `aria-describedby`로 버튼에 연결해 화면 낭독기와 키보드 사용자가 입력 차단 이유를 알 수 있게 한다.
- 업그레이드 행의 상태 문구 영역은 고정 높이로 예약해 구매 가능, 비용 부족, 전환, 구매 성공, 구매 실패 상태가 바뀌어도 행 높이와 모바일 하단 시트의 내부 스크롤 위치가 흔들리지 않게 한다.
- 전환 중 구매 버튼에 포커스가 남아 있어도 Enter, Space, 클릭, 탭은 구매 성공 또는 실패 트랜잭션을 시작하지 않는다.
- 전환이 끝난 뒤에도 전환 중 입력은 자동 재실행되지 않으며, 플레이어가 새 입력을 다시 해야 한다.
- 포커스 가능한 UI 버튼은 Enter 또는 Space로 활성화할 수 있어야 하며, 이때 Space는 고블린 공격으로 새지 않아야 한다.
- 게임 루트 또는 전투 화면에 포커스가 있을 때만 Space `keydown`을 직접 공격 후보로 처리한다.
- 전투 포커스 상태에서 Space `keydown`을 직접 공격 후보로 처리할 때만 `preventDefault()`로 페이지 스크롤을 막는다.
- Space `keyup`과 `event.repeat`이 `true`인 Space `keydown`은 직접 공격으로 처리하지 않는다.
- Escape는 상점 닫기 단축키로 사용하지 않는다.
- Escape는 저장 초기화 확인 모달처럼 닫아도 유효한 이전 상태로 돌아갈 수 있는 취소 가능 모달에서만 `취소`와 동일하게 동작한다.
- 로드 오류 차단 모달에서는 Escape가 모달을 닫거나 암묵적으로 `취소`를 실행하지 않아야 한다.
- 확인 모달이 열린 동안에는 Space와 Enter가 모달 UI에만 적용되고 배경 전투 입력으로 처리되지 않아야 한다.
- 상점, 버튼, 모달 위 입력은 전투 영역으로 포커스를 훔치지 않아야 한다.
- 모바일 상점 시트를 닫으면 하단 `상점` 버튼으로 포커스를 돌린다.
- 저장 초기화 확인 모달이나 로드 오류 모달이 닫히면 모달을 연 버튼 또는 모달 진입 전 포커스 요소로 포커스를 돌린다.
- 로드 오류 상태처럼 게임이 시작되지 않은 상태에서는 전투 영역에 포커스를 주지 않는다.
- 모달 내부를 제외하고 Tab 키 이동을 가로막는 포커스 트랩을 만들지 않는다.
- 모바일 상점 시트 내부에는 상단 우측에 명시적인 `닫기` 버튼을 둔다.
- 모바일 상점이 열린 동안에도 하단 `상점` 토글 버튼은 계속 노출한다.
- 모바일 상점 시트 내부 `닫기` 버튼과 하단 `상점` 토글 버튼은 같은 닫기 동작을 수행하고, 닫힌 뒤 포커스는 하단 `상점` 토글 버튼으로 돌아가야 한다.
- 색상만으로 구매 가능 여부를 표현하지 않는다.
- 모바일의 접힌 상점 버튼은 구매 가능 상태를 구매 가능 개수 배지로 표현하고, 색상만으로 표현하지 않는다.
- 모바일의 접힌 상점 버튼 접근성 이름 또는 설명에는 `구매 가능 업그레이드 2개`처럼 실제 구매 가능 개수가 포함되어야 한다.
- 모바일 하단 `상점` 토글 버튼은 상점 시트와 연결된 `aria-controls`를 가져야 한다.
- 모바일 하단 `상점` 토글 버튼은 상점이 닫혀 있으면 `aria-expanded="false"`, 열려 있으면 `aria-expanded="true"`를 제공해야 한다.
- 모바일 하단 `상점` 토글 버튼의 접근성 이름은 닫힌 상태에서 `상점 열기, 구매 가능 업그레이드 N개`, 열린 상태에서 `상점 닫기, 구매 가능 업그레이드 N개` 형식을 사용한다. 구매 가능 개수가 0이면 `구매 가능 업그레이드 없음`을 사용한다.
- 모바일 상점을 열면 시트 컨테이너 또는 시트 제목에 일시적으로 포커스를 주고, 이 요소는 `tabindex="-1"` 같은 방식으로 프로그램 포커스만 받을 수 있게 한다.
- 모바일 상점 시트 컨테이너 또는 제목에 포커스가 있을 때 Enter와 Space는 구매나 닫기 동작을 실행하지 않아야 한다.
- 모바일 상점 시트를 연 뒤 Tab 이동으로 시트 내부 `닫기` 버튼과 업그레이드 버튼에 접근할 수 있어야 하며, 시트 내부에 Tab 포커스 트랩을 만들지 않는다.
- 체력과 재화는 숫자로도 표시한다.
- 피해 숫자는 체력 바와 주요 상점 버튼을 가리지 않는 위치에서 표시되어야 한다.
- 모바일 하단 시트의 기본 최대 높이는 `45dvh`이며, 내용이 넘치면 내부 스크롤로 처리한다.
- 모바일 하단 시트의 스크롤 컨테이너에는 `padding-bottom`과 `scroll-padding-bottom`을 모두 `하단 상점 토글 높이 + safe-area-inset-bottom + 8px` 이상으로 둔다.
- 모바일 하단 시트의 마지막 업그레이드 행과 마지막 포커스 가능 요소는 스크롤 가능한 최하단 위치와 키보드 포커스 이동 상태 모두에서 하단 `상점` 토글 버튼과 안전 영역에 가려지지 않아야 한다.
- 상점이 열린 상태에서도 모바일 직접 공격 영역은 최소 160px 이상 보여야 하며, 체력 바는 가려지지 않아야 한다.
- 작은 화면에서 `45dvh`와 160px 클릭 영역 보존이 충돌하면, 클릭 영역 보존을 우선하고 시트 높이를 더 줄인다.
- 모바일에서 직접 공격 영역은 최소 `160px x 160px` 이상이며, 레벨 변화, 장식, 피격 애니메이션, squash/stretch로 줄어들거나 이동하지 않아야 한다.
- 스프라이트가 160px보다 크게 보이는 화면에서는 직접 공격 영역이 스프라이트의 보이는 주요 몸통보다 작아지지 않아야 한다.
- 고블린의 영구 스케일과 클릭 판정은 레벨과 무관하게 유지한다.
- 모바일 멀티터치와 키보드 반복 입력은 직접 공격 중복 처리로 이어지지 않아야 한다.
- 중요한 텍스트는 작은 화면에서도 줄바꿈되어야 하며 버튼 밖으로 넘치면 안 된다.

## 16. 테스트 계획

### 필수 단위 테스트

수식과 규칙은 단위 테스트로 검증한다. MVP 완료 게이트에 포함된다.

- `currentGoblinLevel = defeatedCount + 1` 계산
- `currentGoblinMaxHp = calculateGoblinMaxHp(currentGoblinLevel)` 계산
- 현재 고블린 레벨별 기본 처치 보상 계산
- 레벨 0 첫 구매 비용과 구매 전 레벨 기준의 다음 업그레이드 비용 계산
- 낡은 몽둥이 레벨이 반영된 클릭 피해 계산
- 고블린 미끼 주머니 레벨이 반영된 처치 보상 계산
- `mudTrapArmedLevel` 기준으로 첫 직접 공격 배율 적용
- 전투 중 진흙 함정을 추가 구매해도 현재 고블린의 `mudTrapArmedLevel`이 바뀌지 않는지 검증
- 직접 공격으로 `진흙 함정`만 소모되고 고블린이 살아 있으면 `mudTrapArmedLevel`이 `0`인 현재 전투 상태를 1회 저장하는지 검증
- 같은 직접 공격으로 `진흙 함정` 소모와 고블린 처치가 함께 발생하면 중간 상태를 저장하지 않고 다음 고블린까지 확정된 최종 `SaveData`만 1회 저장하는지 검증
- 저장소 접근 실패로 인한 비저장 세션에서도 직접 공격, 체력 감소, 처치, 재화 획득, 다음 고블린 상태 확정이 메모리 상태에서 정상 처리되는지 검증
- 투석기 자동 공격이 진흙 함정 보너스를 받거나 소모하지 않는지 검증
- 시작 상태에서 투석기 레벨이 0이고 `catapultCooldownRemainingMs`가 `0`인지 검증
- 투석기 레벨 0에서는 `catapultCooldownRemainingMs`가 `0`이고 쿨다운이 진행되지 않는지 검증
- 전투 중 투석기 첫 구매 시 `catapultCooldownRemainingMs`가 `5000`으로 설정되는지 검증
- 투석기 추가 구매 시 쿨다운이 리셋되지 않고 다음 자동 공격 피해량만 새 레벨 기준으로 갱신되는지 검증
- 투석기 레벨과 현재 클릭 피해량이 반영된 다음 자동 공격 피해 계산
- 투석기 자동 공격 피해가 진흙 함정 첫 직접 공격 배율을 포함하지 않는지 검증
- 투석기 쿨다운 만료 후 자동 공격 1회 적용과 5000ms 재설정
- 투석기 쿨다운 경과 시간이 `performance.now()` 기반 delta로 계산되는지 검증
- `Date.now()`나 시스템 시각 변경이 투석기 쿨다운, 처치 보상, 체력 계산에 영향을 주지 않는지 검증
- 프레임 경과 시간이 남은 투석기 쿨다운을 초과해도 자동 공격은 1회만 처리하고 초과 시간을 다음 쿨다운에 이월하지 않는지 검증
- 투석기 자동 공격 처치 시 직접 공격 처치와 동일한 처치 수 증가, 다음 고블린 레벨 파생, 보상 지급 적용
- 투석기 자동 공격 처치와 10초 자동 저장 타이머가 같은 업데이트 흐름에서 겹치면 처치 수, 재화, 다음 고블린 상태, 투석기 쿨다운까지 확정한 최종 `SaveData`만 1회 저장하는지 검증
- 같은 프레임에 직접 공격 입력과 투석기 자동 공격 만료가 겹치면 직접 공격을 먼저 처리하는지 검증
- 같은 프레임의 직접 공격으로 고블린이 처치되면 해당 프레임에서 투석기 쿨다운을 차감하지 않고 자동 공격을 발사하거나 다음 고블린에게 지연 예약하지 않는지 검증
- 같은 프레임의 직접 공격 후 고블린이 살아 있으면 그다음 투석기 쿨다운 갱신과 자동 공격 만료 처리를 수행하는지 검증
- `document.visibilityState === "visible"`이면 브라우저 창 포커스가 없어도 투석기 쿨다운이 흐르는지 검증
- `document.visibilityState === "hidden"`이면 투석기 쿨다운이 멈추는지 검증
- hidden에서 visible로 돌아왔을 때 hidden 동안의 경과 시간을 투석기 쿨다운에 반영하지 않고 `performance.now()` 기준 시간을 새로 잡는지 검증
- 피해 적용 후 `GameState.goblinHp`가 `max(0, goblinHp - damage)`로 클램프되고 음수가 되지 않는지 검증
- 런타임 피해 처리 중 `goblinHp`가 0이 될 수 있지만, 이 상태가 현재 고블린의 저장 가능한 상태로 남지 않는지 검증
- 직접 공격과 투석기 자동 공격이 같은 순간 발생해도 피해 숫자를 합산하지 않고 별도로 생성하는지 검증
- 피해 숫자가 최대 `6개`까지만 동시에 표시되고 초과 시 가장 오래된 숫자부터 제거되는지 검증
- 피해 숫자가 약 `600ms` 후 제거되는지 검증

- 첫 실행 시작 상태 저장이 실패해도 게임을 시작하고 HUD에 `저장되지 않음` 경고를 표시하며 다음 저장 트리거에서 다시 시도하는지 검증
- 앱 시작 시 `localStorage` 읽기 자체가 실패하면 이를 저장 데이터 없음으로 처리하지 않고 저장소 접근 실패로 인한 비저장 세션으로 표시하는지 검증
- 저장소 접근 실패로 인한 비저장 세션에서는 시작 상태를 `goblin-clicker.save`에 저장하지 않고, 10초 자동 저장, 처치 저장, 구매 저장도 실제 쓰기를 시도하지 않는지 검증
- 저장소 접근 실패로 인한 비저장 세션에서 HUD에 `저장소에 접근할 수 없어 저장되지 않음` 수준의 경고가 표시되는지 검증
- 저장소 접근 실패로 인한 비저장 세션에서 새로고침하면 현재 세션 진행이 사라질 수 있음을 HUD가 명확히 알리는지 검증
- 10초 자동 저장이 게임 진행 타이머가 아니라 저장 보조 트리거로 동작하는지 검증
- 10초 자동 저장이 지연되거나 hidden에서 재개되어도 밀린 저장을 여러 번 몰아서 실행하지 않고 다음 자동 저장 시도 1회만 수행하는지 검증
- 로드 오류 상태에서는 10초 자동 저장을 실행하지 않는지 검증
- 게임이 시작된 세션에서 저장할 `SaveData`가 있을 때만 10초 자동 저장을 시도하는지 검증
- 10초 자동 저장 실패 시 현재 세션 플레이를 유지하고 다음 저장 트리거에서 다시 시도하는지 검증
- 마지막 저장 시도가 실패하면 HUD에 `저장되지 않음` 경고가 표시되고, 이후 어떤 저장 트리거든 `goblin-clicker.save` 쓰기에 성공하면 즉시 해제되는지 검증
- 저장 데이터 복원 시 가장 먼저 `SaveData.saveVersion`을 확인하고, `saveVersion: 1`인 경우에만 필드별 정규화를 수행하는지 검증
- 저장 데이터 복원 시 `defeatedCount`와 `coins`가 유효한 0 이상의 안전한 정수가 아니면 0으로 정규화하는지 검증
- 저장 데이터 복원 시 `currentGoblinLevel`과 `currentGoblinMaxHp`를 정규화된 `defeatedCount`에서 파생하는지 검증
- 저장 데이터 복원 시 유효한 `goblinHp` 범위가 `1..currentGoblinMaxHp`이고, `goblinHp === 0`은 `currentGoblinMaxHp`로 정규화하는지 검증
- 저장 데이터 복원 시 누락, `NaN`, `Infinity`, 음수, 소수, 숫자가 아닌 `goblinHp`는 `currentGoblinMaxHp`로 정규화하는지 검증
- 저장 데이터 복원 시 `upgrades`가 MVP 4종 키만 보존하고, 누락 키는 0으로 채우며, 알 수 없는 키는 무시하는지 검증
- 저장 데이터 복원 시 업그레이드 레벨이 유한한 0 이상의 안전한 정수가 아니면 0으로 정규화하는지 검증
- 저장 데이터 복원 시 투석기 레벨이 0이면 `catapultCooldownRemainingMs`를 0으로, 투석기 레벨이 1 이상이면 유효값은 `0..5000`으로 클램프하고 유효하지 않은 값은 `5000`으로 정규화하는지 검증
- 저장 데이터 복원 후 `performance.now()` 값을 복원하지 않고 새 세션 기준 시간으로 투석기 delta를 계산하는지 검증
- 저장 데이터 복원 시 `mudTrapArmedLevel`을 `0..현재 진흙 함정 레벨` 범위로 클램프하고 유효하지 않은 값은 0으로 정규화하는지 검증
- 필드별 정규화로 복원 가능한 저장 데이터는 사용자 경고 없이 게임을 계속 시작하는지 검증
- 저장 데이터 파싱 실패와 마이그레이션 실패 시 로드 오류 상태로 전환하고 차단 모달을 표시하는지 검증
- `localStorage` 읽기 실패가 저장 데이터 파싱 실패나 마이그레이션 실패와 다른 상태로 처리되는지 검증
- 로드 오류 모달 문구가 새 게임 시작 시 기존 저장 삭제 결과를 명확히 알리는지 검증
- 로드 오류 모달에서 `새 게임 시작`을 선택하면 `goblin-clicker.save` 값을 삭제하고 시작 상태로 새 게임을 시작한 뒤 시작 상태를 저장하는지 검증
- 로드 오류 모달의 `새 게임 시작`에서 `goblin-clicker.save` 삭제는 성공했지만 시작 상태 저장이 실패하면 새 게임 세션을 계속 시작하고 `저장되지 않음` 경고를 표시하며 다음 저장 트리거에서 다시 시도하는지 검증
- 로드 오류 모달의 `새 게임 시작`에서 `goblin-clicker.save` 삭제가 실패하면 기존 저장을 유지한 것으로 간주하고 새 게임을 시작하지 않으며 `저장 데이터를 삭제할 수 없습니다.` 경고와 다시 시도 또는 취소 선택지를 유지하는지 검증
- 로드 오류 모달에서 `취소`를 선택하면 `goblin-clicker.save` 값을 삭제하지 않고 새 게임도 시작하지 않으며 로드 오류 상태에 머무는지 검증
- 로드 오류 차단 모달에서 Escape를 눌러도 `취소` 버튼이 실행되지 않고, 기존 저장 삭제나 새 게임 시작 없이 로드 오류 상태에 머무는지 검증
- 로드 오류 모달에서 `다시 시도`를 선택하면 현재 `localStorage` 값을 다시 읽고 전체 로드 파이프라인을 처음부터 재실행하는지 검증
- 다시 시도가 성공하면 로드 오류 모달을 닫고 복원된 게임을 시작하는지 검증
- 다시 시도가 실패하면 기존 저장을 삭제하지 않고 같은 로드 오류 상태와 차단 모달을 유지하는지 검증
- 로드 오류 상태에서 저장하지 않는 임시 플레이나 자동 저장 덮어쓰기가 발생하지 않는지 검증
- 진흙 함정 준비 레벨, 소모 상태, 투석기 쿨다운 남은 시간 복원
- `localStorage` 쓰기 실패 시 현재 세션 플레이 유지와 저장 실패 상태 표시
- 저장 실패 HUD 경고가 로드 오류, 저장 데이터 삭제 실패, 파싱 실패, 마이그레이션 실패 같은 차단 오류를 대신 표시하지 않는지 검증
- 저장 초기화 확인 시 `goblin-clicker.save` 값 삭제와 시작 상태 재초기화 및 시작 상태 저장
- 저장 초기화에서 `goblin-clicker.save` 삭제는 성공했지만 시작 상태 저장이 실패하면 새 게임 세션을 계속 진행하고 `저장되지 않음` 경고를 표시하며 다음 저장 트리거에서 다시 시도하는지 검증
- 저장 초기화에서 `goblin-clicker.save` 삭제가 실패하면 기존 저장을 유지한 것으로 간주하고 시작 상태로 재초기화하지 않으며 `저장 데이터를 삭제할 수 없습니다.` 경고와 다시 시도 또는 취소 선택지를 유지하는지 검증
- 저장소 접근 실패로 인한 비저장 세션에서는 저장 초기화 버튼이 실제 비활성화되거나, 입력해도 상태를 바꾸지 않고 `저장소에 접근할 수 없어 초기화할 수 없습니다` 경고만 표시하는지 검증
- 비저장 세션에서 저장 초기화를 시도해도 `goblin-clicker.save` 삭제, 시작 상태 재초기화, 시작 상태 저장이 발생하지 않는지 검증
- 비저장 세션에서 현재 세션만 로컬로 초기화하는 기능이 제공되지 않는지 검증
- 저장 초기화 취소 버튼 또는 Escape 입력 시 상태 미변경
- 저장 메타데이터 `savedAt`이 게임 진행 계산에 영향을 주지 않는지 검증
- `Date.now()`나 시스템 시각이 저장 메타데이터 표시 외의 게임 진행 계산에 영향을 주지 않는지 검증
- `SaveData.saveVersion` 1 저장 데이터 복원
- `SaveData.saveVersion` 누락 데이터는 레거시 저장 데이터로 분류하지만 MVP에서 지원하는 레거시 포맷이 없으므로 마이그레이션 실패로 처리하는지 검증
- 지원하지 않는 `SaveData.saveVersion`의 마이그레이션 실패 처리

### 핵심 브라우저 스모크 테스트

실제 사용자 흐름은 6~8개의 핵심 스모크 테스트로 제한한다. MVP 완료 게이트에 포함된다.

- 첫 고블린을 5번 클릭하면 처치 수가 1이 되고 재화가 증가한다.
- 2번째 고블린 처치 직후 `낡은 몽둥이`를 첫 업그레이드로 구매할 수 있고, `낡은 몽둥이`에는 작은 `추천` 표시가 보이며, 다른 업그레이드는 상점에 보이지만 비용 부족 상태다.
- `낡은 몽둥이`를 첫 구매하면 `추천` 표시가 제거된다.
- 상점 업그레이드 행은 `낡은 몽둥이`, `삐걱대는 투석기`, `고블린 미끼 주머니`, `진흙 함정` 순서로 표시되며 구매 가능 상태가 바뀌어도 순서가 바뀌지 않는다.
- 재화가 충분할 때 업그레이드를 한 번에 1레벨 구매할 수 있고, 구매 후 클릭 피해량이 증가한다.
- 업그레이드 버튼을 빠르게 연속 입력해도 각 입력은 최신 재화와 다음 비용 기준으로 순차 처리된다.
- 새로고침 후 저장된 처치 수와 재화가 복원된다.
- 모바일에서 상점은 기본 접힘 상태이며, 실제 구매 가능 업그레이드가 있으면 접힌 상점 버튼에 `상점 1`, `상점 2` 같은 구매 가능 개수 배지가 보인다.
- `낡은 몽둥이`가 추천 상태여도 재화가 부족하면 접힌 상점 버튼에는 구매 가능 표시가 보이지 않는다.
- 모바일 하단 `상점` 토글 버튼은 상점 열림 여부에 따라 `aria-expanded`와 접근성 이름이 갱신된다.
- 모바일 상점을 열면 시트 컨테이너 또는 시트 제목으로 포커스가 이동하고, Enter나 Space가 즉시 구매나 닫기를 실행하지 않는다.
- 모바일 상점이 열린 상태에서도 시트 내부 상단 우측 `닫기` 버튼과 하단 `상점` 토글 버튼이 둘 다 보이고, 둘 중 어느 쪽으로 닫아도 하단 `상점` 토글 버튼으로 포커스가 돌아온다.
- 모바일 상점을 열면 하단 시트의 기본 최대 높이는 `45dvh`이고, 내용이 넘치면 내부 스크롤된다.
- 모바일 상점 시트의 마지막 업그레이드 행과 마지막 포커스 가능 요소는 스크롤 최하단과 키보드 이동 상태에서 하단 `상점` 토글 버튼이나 안전 영역에 가려지지 않는다.
- 작은 모바일 화면에서 상점을 열어도 직접 공격 영역은 최소 160px 이상 보이고 체력 바가 가려지지 않는다.
- 모바일 viewport에서 고블린, 체력 바, 상점 버튼, 열린 상점 시트가 판독 불가능하게 겹치지 않는다.

### 추가 브라우저 회귀 테스트

시드 상태나 테스트 헬퍼가 필요한 세부 통합 규칙은 추가 브라우저 회귀 테스트로 유지한다. MVP 규칙의 장기 회귀 방지용이며, MVP 완료 게이트에는 포함하지 않는다.

- 80ms 안에 연속 입력된 고블린 공격은 한 번만 처리된다.
- 직접 공격 영역 안에서 시작한 `pointerdown`은 즉시 직접 공격 후보가 되지만 후속 `click`은 공격으로 처리되지 않는다.
- 진흙 함정 구매 후 새 고블린의 첫 번째 직접 공격 피해에 레벨별 배율이 적용된다.
- 진흙 함정이 준비된 현재 고블린에서 추가 구매해도 첫 직접 공격 배율은 고블린 등장 시점의 준비 레벨을 따른다.
- 진흙 함정 소모 후 새로고침해도 같은 고블린에서 첫 직접 공격 보너스가 다시 생기지 않는다.
- 투석기 구매 후 다음 자동 공격까지 남은 시간과 피해량이 표시된다.
- 투석기 레벨 0에서는 HUD에 투석기 상태가 표시되지 않는다.
- 투석기 추가 구매 후 쿨다운 표시는 리셋되지 않고, 다음 자동 공격 피해량 표시만 새 레벨 기준으로 갱신된다.
- 투석기 쿨다운 만료 후 자동 공격은 한 번만 적용되고, 초과 시간 이월 없이 다음 쿨다운이 `5000ms`로 다시 표시된다.
- 오버킬 피해 숫자는 실제 가한 총 피해량을 표시하되, 다음 고블린은 최대 체력으로 등장한다.
- 오버킬이 발생해도 체력 숫자와 즉시 체력 바는 0 미만으로 표시되지 않는다.
- 피해 숫자가 최대 `6개`까지만 동시에 표시되고, 투석기 피해 숫자는 직접 공격 피해 숫자와 구분된다.
- 모바일에서 피해 숫자가 체력 바와 주요 상점 버튼을 가리지 않는다.
- 피해를 받으면 실제 체력 숫자와 즉시 체력 바는 바로 줄고, 잔상 바만 약 `250ms` 늦게 따라온다.
- 다음 고블린 등장 시 실제 체력 바와 잔상 바가 모두 새 고블린의 최대 체력으로 표시된다.
- 처치 후 300ms 전환 시간 동안 다음 고블린 표시가 지연되고, 전환 중 입력이 다음 고블린 공격으로 새지 않는다.
- 처치 후 300ms 전환 중 처치 수와 보유 재화는 즉시 갱신되지만, 고블린 레벨과 체력 바는 다음 고블린 등장 시점에 새 값으로 갱신된다.
- 처치 후 300ms 전환 중 새로고침하면 전환 연출을 복원하지 않고 다음 고블린이 즉시 표시된다.
- 처치 후 300ms 전환 중 상점은 보일 수 있지만 구매 버튼은 포커스 유지형 입력 차단 상태가 되고, 구매 입력은 다음 고블린 등장 후 자동 처리되지 않는다.
- 처치 후 300ms 전환 중 구매 버튼에는 `잠시 후 구매 가능` 또는 `처치 처리 중` 같은 버튼 단위 상태 텍스트가 표시된다.
- 처치 후 300ms 전환 중 비용 부족 업그레이드 버튼을 눌러도 비용 부족 실패 피드백이 표시되지 않는다.
- 처치 후 300ms 전환 중 구매 버튼 포커스는 가능하면 유지되고, 전환 중 Enter나 Space 입력은 전환 종료 후 자동 재실행되지 않는다.
- 모바일 상점이 열린 상태에서 직접 공격이나 투석기 자동 공격으로 처치 전환이 시작되어도 하단 시트의 열린 상태와 내부 스크롤 위치가 유지된다.
- 모바일 상점에서 업그레이드 상태 문구가 바뀌어도 업그레이드 행 높이가 변하지 않고, 사용자가 직접 스크롤하지 않는 한 내부 스크롤 위치가 유지된다.
- 상점이 열린 동안에도 투석기 쿨다운과 자동 공격이 진행된다.
- 모바일 상점에서 구매 후 상점은 열린 상태를 유지한다.
- 업그레이드 구매 성공 시 버튼과 숫자 중심의 로컬 피드백만 표시되고 전역 축하 모달, 화면 전체 연출, 긴 튜토리얼 안내가 뜨지 않는다.
- 비용 부족 업그레이드 버튼을 누르면 해당 버튼에만 실패 피드백이 표시되고 전역 모달이나 토스트 큐가 뜨지 않는다.
- 비용 부족 업그레이드 버튼은 키보드 포커스를 받을 수 있고 Space와 Enter 입력 시 구매 실패 피드백만 표시된다.
- 모바일 상점이 열린 동안 시트 밖 직접 공격 영역에서는 공격할 수 있고, 시트 위 입력은 고블린 공격으로 새지 않는다.
- 모바일 직접 공격 영역은 최소 `160px x 160px` 이상이며, 스프라이트가 크게 보일 때 보이는 주요 몸통보다 작아지지 않는다.
- 상점, 버튼, 확인 모달 위 입력은 직접 공격 영역과 겹쳐도 고블린 공격으로 처리되지 않는다.
- 구매 버튼, 닫기 버튼, 확인 모달 버튼에 포커스가 있을 때 Space와 Enter가 고블린 공격으로 새지 않는다.
- 확인 모달이 열린 동안 Space가 배경 고블린 공격으로 처리되지 않는다.
- 로드 오류 차단 모달이 열린 동안 배경 고블린 공격과 업그레이드 구매 입력이 처리되지 않는다.
- 모바일 상점은 명시적 닫기 버튼이나 상점 토글로만 닫힌다.
- Escape는 상점을 닫지 않고, 저장 초기화 확인 모달에서는 취소로 동작하며, 로드 오류 차단 모달에서는 무시된다.
- 저장 실패가 발생해도 현재 세션에서 고블린 공격과 업그레이드 구매를 계속할 수 있다.
- 저장 실패 경고가 표시된 뒤 저장이 성공하면 HUD의 `저장되지 않음` 경고가 사라진다.
- 저장 데이터가 없는 첫 실행에서 시작 상태 저장이 실패해도 전투 화면이 시작되고 `저장되지 않음` 경고가 표시된다.
- 저장소 접근 실패로 인한 비저장 세션에서도 전투 화면은 시작되지만 저장 트리거가 실제 쓰기를 시도하지 않고, HUD에 저장소 접근 실패 경고가 표시된다.
- 저장소 접근 실패로 인한 비저장 세션에서도 고블린 처치, 재화 획득, 업그레이드 구매는 메모리 상태에서 정상 동작한다.
- 저장소 접근 실패로 인한 비저장 세션에서는 저장 초기화가 실행되지 않고, 현재 세션만 로컬로 초기화할 수도 없다.
- 저장 초기화 취소 시 진행 상태가 유지되고, 확인 시 시작 상태로 돌아간다.

## 17. MVP 완료 기준

- 고블린 클릭, 체력 감소, 처치, 다음 고블린 등장 흐름이 동작한다.
- 재화 획득과 업그레이드 구매가 동작한다.
- MVP 업그레이드 4종이 구현되어 있다.
- 게임 상태가 저장되고 새로고침 후 복원된다.
- 데스크톱과 모바일에서 주요 UI가 겹치지 않는다.
- 필수 단위 테스트와 핵심 브라우저 스모크 테스트가 통과한다.
- 첫 10분 또는 대략 40마리 처치까지의 밸런스 목표를 검증한다. 그 이후의 장시간 체력, 보상, 비용 곡선 정교화와 콘텐츠 새로움은 MVP 완료 기준 밖이다.

## 18. MVP 이후 확장 후보

- 고블린 종류별 외형과 특성
- 고블린 실루엣 변화와 크기 변화
- 보스 고블린
- 엔딩 또는 레벨 상한
- 리셋 보상 또는 프레스티지
- 오버킬 피해 이월, 관통 피해, 연쇄 처치
- 겁쟁이 북, 녹슨 왕관 감별기, 용병 계약서 같은 추가 업그레이드
- `x10 구매`, 최대 구매, 구매 배수 선택
- 오프라인 보상
- 다중 저장 슬롯, 버전별 저장 키, 백업 키
- 저장하지 않고 임시 플레이, 클라우드 백업, 파일 다운로드 백업, 수동 복구 UI
- 저장소 접근 실패 재시도, 브라우저 권한 안내, 저장소 접근 복구 UI
- 정규화 알림 UI, 저장 진단 화면, 복구 상세 리포트
- 모바일 상점 스와이프 다운 닫기
- 업그레이드 초기화와 영구 보너스
- 업적
- 치명타 시스템
- 짧은 퀘스트
- 시즌별 스킨
- 픽셀 아트 스킨

이 항목들은 MVP 범위가 아니다. 첫 구현에서는 핵심 클릭 루프와 업그레이드 루프를 완성한 뒤 판단한다.
